# AutoJS 自动化

AutoJS 自动化（autojs-automation，兼容 slug: ajs）负责本技能描述范围内的定位、执行、验证和交接边界；旧短 slug 仅作兼容 alias/URL 主键，不作为规范技能名。

> 定位：把 Auto.js / AutoX.js / AutoJS Pro 自动化从“脚本能跑”收敛成可复核排障闭环：版本/设备/权限/入口/证据 → 场景执行卡 → 高频坑/防遗漏 → 输出要求 → 约束 → 反例库 → 提交前自检 → 2024-2026 新坑速查 → 相邻技能边界。
> 铁律：合法自有设备、授权测试、无障碍辅助优先；无版本、设备、权限、控件树、截图或稳定性日志证据，不宣称已修复；严禁恶意刷量、绕风控、账号滥用、抢购作弊、规避平台规则、窃取数据或攻击第三方 App。

## 快速总则

### 单技能工程门禁

- 先判定是否合法：只允许自有设备、授权测试、无障碍辅助、企业内控巡检和设备运维；批量账号、刷量、抢购、绕风控、验证码/实名/支付绕过直接拒绝。
- 先锁定运行画像：设备、ROM、Android、目标 App、AutoJS 引擎、权限、分辨率、字体缩放、网络、登录态和页面入口未明确，不得宣称脚本可稳定运行。
- 先定义成功/失败判据：每个 step 都要有进入条件、动作、业务信号等待、超时、重试上限、失败证据、人工接管和可恢复状态。
- 先做幂等保护：涉及提交、删除、支付确认、发布、发消息、改资料等副作用动作时，必须有唯一键、二次确认、结果回读和不确定即停。
- 先做真机矩阵：至少覆盖目标最低 Android、主力厂商 ROM、目标分辨率/字体、深浅色、弱网、锁屏/前后台和权限弹窗；未测设备只列风险。
- 先做证据闭环：运行日志、截图/录屏、控件树摘要、选择器、bounds、Activity、runId、设备和异常堆栈必须能复盘。
- 先做合规边界：所谓反检测只能用于降低误触、误封、误报和测试噪声；不得提供绕过平台风控、验证码、实名、支付确认、账号限制、反作弊或安全检测的实现路径。

### 低级错禁止清单

- 禁止固定 sleep 代替 waitFor 业务信号。
- 禁止绝对坐标作为首选路径；坐标兜底必须有区域校验、比例换算和动作后确认。
- 禁止 findOne 无超时、无限滑动、无限重试、高频截图/OCR 或死循环保活。
- 禁止只测模拟器、云手机或单台旗舰机就说兼容真机矩阵。
- 禁止未检查无障碍/截图/悬浮窗/通知/前台服务/电量优化就改 selector。
- 禁止把 OCR、找图、旧截图或控件树空结果当确定事实。
- 禁止日志写入账号、手机号、token、cookie、身份证、未脱敏截图或敏感 OCR 文本。
- 禁止脚本失控后继续点击；离开目标 package/activity、出现系统设置/支付/授权页时必须停下或人工接管。

### 真实运行闭环验收

- 启动闭环：脚本版本、配置版本、资源版本、目标 package/activity、权限状态、首屏控件、登录态和停止入口全部记录。
- 动作闭环：每次点击/输入/滑动/OCR 都记录 step、selector/region、bounds、前后截图或业务信号、耗时、重试和结果。
- 异常闭环：权限缺失、控件不存在、页面漂移、网络慢、App 崩溃、脚本异常、系统回收、OCR 低置信分别给恢复路径。
- 发布闭环：单设备灰度、矩阵复测、上一版脚本/配置/资源回滚、运行日志样例、无法验证设备清单和人工接管说明。
- 安全闭环：截图、OCR、日志和导出文件默认脱敏；账号/支付/隐私/风控场景只给合规测试与拒绝口径。

1. 版本先钉住：记录 Auto.js、AutoX.js、AutoJS Pro、V8/Rhino/V9 Node 引擎、Android 版本、targetSdk、目标 App 版本、WebView 版本、ADB/打包工具链；版本坑不清只列补证，不直接改 API。
2. 设备先钉住：真机品牌、型号、SoC、Android 12/13/14/15、厂商 ROM、分辨率、DPI、横竖屏、刘海/挖孔/导航栏、字体缩放、深色模式、多窗口/折叠屏状态必须进入排障记录。
3. 权限先钉住：无障碍服务、悬浮窗权限、截图权限、前台服务通知、自启动、后台弹出界面、电池优化、存储/媒体权限、通知权限逐项确认；权限缺失不是 selector 失效。
4. 入口先钉住：脚本入口、目标 package/activity、页面起点、账号/登录态、网络、弹窗、任务来源、停止按钮、回滚方式和人工接管路径必须明确。
5. 证据先钉住：每个结论绑定复现步骤、脚本日志、截图/录屏、控件树、selector、bounds、当前包名、当前 Activity、时间戳、设备与系统版本。
6. 控件优先：优先使用 id/text/desc/className/packageName/boundsInside/depth/indexInParent 等组合选择器；坐标只做兜底，必须加区域校验和多分辨率适配。
7. 等待优先业务信号：点击、滑动、输入、返回后等待控件、文本、Activity、截图特征、列表状态或网络结果；禁止固定 sleep 冒充 waitFor。
8. 截图/OCR 只做感知证据：captureScreen、找图、OCR 必须处理授权弹窗、旧帧、旋转、缩放、颜色模式、隐私遮挡；OCR 结果不能替代稳定控件 selector。
9. 稳定性日志必须可追溯：记录 step、selector、bounds、action、result、duration、retry、screenshotId、package/activity、battery、foreground/background、异常堆栈。
10. 平台差异必须写清：Auto.js 4.x、AutoX.js、AutoJS Pro、不同 JS 引擎、不同 Android API、不同厂商 ROM 的权限、后台、截图、WebView 控件树表现不能混为一谈。

## 场景执行卡

### 1. 新建或迁移脚本
- 先确认目标：合法测试、无障碍辅助、个人效率工具、企业内控巡检还是设备运维；涉及刷量、风控规避、账号批量操作直接拒绝。
- 建脚本骨架前先列设备矩阵、权限矩阵、目标 App 版本、入口页面、成功判据、失败判据、停止/回滚入口。
- Auto.js 到 AutoX.js/AutoJS Pro 迁移时，核对模块 API、线程模型、引擎差异、打包行为、权限声明、Node/V8 兼容和第三方库可用性。
- 交付最小闭环：启动校验、权限校验、主流程、异常分支、超时退出、日志、截图证据、人工接管。

### 2. 无障碍服务与权限排障
- 先确认无障碍服务是否已启用、是否被系统回收、是否绑定正确包名、是否具备 observeKey、悬浮窗和截图权限。
- Android 12-15 上重点检查前台服务限制、通知权限、后台启动限制、精确闹钟、媒体权限、截图授权弹窗和厂商电量优化。
- MIUI、ColorOS、OriginOS、HarmonyOS/EMUI、OneUI 对自启动、后台弹窗、锁屏运行、悬浮窗层级、无障碍保活策略不同，必须分设备记录。
- 权限失败要输出系统设置路径、截图、当前状态和复测步骤，不把权限问题包装成代码 bug。

### 3. 控件选择器定位与点击失效
- 先抓控件树、当前 package/activity、目标节点 text/id/desc/className/bounds；同名节点必须加父子关系、区域或索引约束。
- findOne 必须设置超时；超时后记录页面截图、控件树摘要、selector、当前 Activity，并进入降级或人工接管。
- 点击前校验 visibleToUser、bounds、enabled/clickable 或父节点 clickable；点击后用页面变化、文本变化或 Activity 变化确认生效。
- WebView、RecyclerView、Compose 页面可能暴露不完整控件树，需要结合滚动、截图特征和业务状态，不盲目追加 sleep。

### 4. 坐标兜底与多分辨率适配
- 坐标只在控件不可访问、图像目标稳定、授权场景明确时使用；必须以设计区域、控件 bounds、屏幕比例、DPI 和安全区域换算。
- 多分辨率适配要覆盖 720p/1080p/1440p、平板、折叠屏、横竖屏、虚拟导航栏、状态栏、字体缩放；不要只用单机绝对坐标。
- 每个坐标动作前做截图或控件区域校验，动作后校验结果；失败要记录触点、屏幕尺寸、缩放系数和截图。
- swipe/scroll 要设置方向、距离、速度、次数上限和终止条件；不能无限滑或误触系统返回/通知栏。
- gesture/press/drag 等手势必须声明起止区域、持续时间、速度范围、目标元素或截图特征；手势前后都要校验页面签名，禁止用连续随机手势冒充真人行为或规避检测。

### 5. 截图、找图与 OCR
- captureScreen 前确认截图权限、前台状态、屏幕常亮、隐私遮挡和旋转；每次截图加时间戳，避免旧图复用。
- 找图要记录模板来源、分辨率、缩放、阈值、匹配区域和失败截图；模板图随 App 版本、主题和字体变化需要复核。
- OCR 要限定区域、语言、置信度、纠错词表和误识别兜底；金额、验证码、敏感内容不做未授权识别或外传。
- OCR/图像识别失败时先比对截图质量、遮挡、主题、缩放和帧延迟，再判断算法或脚本问题。

### 6. WebView、Intent、App 启动与页面切换
- 启动 App 后校验 package/activity、前台窗口、登录态和首屏关键控件；Intent 成功不等于页面可操作。
- WebView 页面要记录 WebView 版本、H5 路由、加载状态、网络状态、控件树可见性；H5 异步加载必须等业务信号。
- 处理弹窗、权限框、升级提示、广告层、登录过期、网络异常和系统设置跳转；弹窗优先按控件识别，不用盲点固定坐标。
- 返回/切换任务后必须确认回到目标 Activity，避免脚本在桌面、系统设置或第三方授权页继续点击。

### 7. 保活、电量优化与任务调度
- 保活不是无限后台运行；要声明前台服务、通知、Work/Alarm/定时策略、厂商白名单、电量优化设置和用户可见停止入口。
- Android 12-15 对后台启动、前台服务类型、精确闹钟、通知权限、任务杀后台更严格；不能承诺系统一定不杀进程。
- 长任务要分 step checkpoint，支持暂停、恢复、幂等、最大运行时长、失败次数上限和人工接管。
- 调度任务必须记录 taskId、runId、nextRunAt、lease/锁、去重键、过期时间和最后心跳；脚本重启、网络恢复、重复拉取或多设备并发时，不得重复执行副作用动作。
- 电量优化要以低频轮询、事件触发、区域截图、日志采样和资源释放为主，不用高频截图/OCR/轮询硬撑稳定性。

### 8. 日志、异常恢复与多设备验证
- 日志至少包含 step、action、selector/region、package/activity、duration、result、retry、error、screenshotId、device、Android、engine。
- 异常处理区分权限缺失、控件不存在、页面错误、网络慢、App 崩溃、脚本异常、系统回收、OCR 低置信；不同根因不同兜底。
- 多设备验证至少覆盖目标最低 Android、主力厂商 ROM、高低分辨率、深色/浅色、字体缩放和网络慢场景。
- 真实验收不得只看“脚本无报错”；必须给出一条成功链路、至少三类失败注入、一次中断恢复、一次重复任务幂等验证和一份可回放证据包。
- 发布脚本要有版本号、变更点、回滚包、权限说明、风险提示、运行日志样例和验收证据。

## 工程化与运行闭环

- 结构：区分 config、selectors、actions、flows、guards、logger、assets；选择器、坐标、模板图、OCR 词表和页面签名集中管理并版本化，账号、token、环境参数和敏感配置不得硬编码进脚本或日志。
- 流程：复杂任务按有限状态机建模，定义进入条件、退出条件、可重入动作、超时、失败转移、幂等保护和人工接管；涉及提交、删除、下单、转账等副作用动作时，不确定结果必须停下补证，不盲目重试。
- 并发：使用 threads、events、timers、悬浮窗按钮或按键监听时，必须定义生命周期、停止信号、异常上报、监听解绑、共享状态保护和资源释放；人工停止触发后所有工作线程应在有限时间内退出。
- 输入：文本输入前校验焦点、字段类型、当前输入法、已有内容和最大长度；输入后读取控件文本或截图区域验证，不把 input/setText 调用成功等同于业务成功；剪贴板仅作兼容兜底，敏感内容用后清理或恢复并脱敏记录。
- 列表：遍历列表、分页或批处理时，必须定义 item 唯一键、已处理集合、滚动终止条件、空列表判据、最大处理数量和成功/跳过/失败/待人工确认四类结果；禁止无限滑动或靠屏幕坐标猜测当前项。
- 数据：每次运行生成 runId，关联脚本版本、配置版本、资源版本、设备、Android、目标 App 版本、关键截图和结果汇总；运行日志、截图、OCR 文本和导出文件必须设置脱敏、访问范围、保留周期和清理策略，默认不上传第三方服务。
- 增强通道：Root、Shizuku、ADB、shell、模拟器或云手机只能用于自有设备、授权测试或设备运维；必须记录授权来源、命令影响面、回滚方式和失败降级，不用于绕过平台风控、权限模型、验证码、支付确认或第三方 App 安全策略。
- 发布：脚本版本、配置版本、资源版本分开管理；高风险变更先单设备灰度，再扩到设备矩阵；每次发布保留上一版脚本、配置和资源回滚入口，兼容声明只基于实测矩阵。

## 高频坑 / 防遗漏

### 高频坑
- 把无障碍服务未开启、被回收、权限被系统撤销误判为 selector 失效。
- 只用 text 选择器，遇到同名按钮、国际化、A/B 文案、空格符号就误点。
- 坐标写死在 1080p，换 DPI、横屏、全面屏导航栏、折叠屏后全偏。
- captureScreen 复用旧帧，OCR 读到上一页或弹窗遮挡内容。
- 固定 sleep 盖住竞态，网络慢、动画慢、WebView 慢时仍然失败。
- 忽略 Android 13 通知权限、Android 14 前台服务限制、Android 15 后台/权限收紧。
- 保活只靠循环和悬浮窗，不处理电量优化、自启动、前台服务和用户停止入口。
- 日志只有“点击成功”，没有 selector、bounds、截图、Activity、耗时和失败原因。
- 把自动化用于刷量、抢购、绕风控、批量账号、采集隐私，越过合法边界。
- 未脱敏输出账号、手机号、token、截图中的隐私信息。

### 防遗漏清单
- 版本：Auto.js/AutoX.js/AutoJS Pro、JS 引擎、Android、目标 App、WebView、厂商 ROM 已记录。
- 权限：无障碍、悬浮窗、截图、通知、前台服务、自启动、后台弹出、电量优化已核对。
- 选择器：id/text/desc/className/bounds/父子关系/可点击性/超时/失败证据已覆盖。
- 坐标：仅兜底，已绑定区域校验、分辨率、DPI、状态栏/导航栏、安全区。
- 截图/OCR：已处理授权、旧帧、区域、阈值、置信度、隐私脱敏和失败截图。
- 稳定性：有重试上限、退出条件、异常分类、人工接管、运行时长限制和资源释放。
- 证据：日志、截图/录屏、控件树摘要、设备矩阵、复测步骤和结论四档齐全。

## 输出要求

- 必须输出：目标场景、合法授权边界、设备/系统/ROM、Auto.js/AutoX.js/AutoJS Pro 版本、目标 App 版本、权限状态、复现步骤、核心证据、结论四档。
- 方案必须给：首选控件 selector、坐标兜底条件、等待条件、超时/重试策略、截图/OCR 证据、异常恢复、人工接管、回滚或停止方式。
- 排障报告必须列：成功路径、失败路径、不可验证项、平台差异、版本坑、设备差异、风险边界和下一步补证。
- 若涉及脚本修改，必须说明影响面、调用入口、兼容设备、测试命令或手测步骤、日志/截图证据；未验证的设备不得写“已兼容”。
- 涉隐私、账号、风控、支付、营销数据时，只输出合规测试和风险拒绝口径；不提供滥用流程、绕过策略或批量化攻击脚本。

- 只服务合法自有设备、授权测试、无障碍辅助、企业内控自动化、设备运维和排障复现。
- 拒绝恶意刷量、绕风控、账号滥用、抢购作弊、验证码/实名/支付绕过、批量注册登录、隐私采集、攻击第三方 App、规避平台规则。
- 不承诺脚本在所有厂商 ROM 后台永久运行；保活只能给合规、用户可见、可停止的策略和证据口径。
- 不把 AutoJS 自动化扩大为移动安全、协议逆向或抓包绕过；相关需求按相邻技能边界转交。
- 不输出明文 token、cookie、手机号、身份证、私钥、账号密码、未脱敏截图或可识别个人信息。
- 无权限、无设备、无复现、无日志时，只能给补证清单和假设，不得宣称根因或已修复。

## 高频 Bug 反例库

- 反例 1：版本未确认就改 API。错法：按 Auto.js 4.x 经验改 AutoX.js 脚本。对法：先记录 Auto.js/AutoX.js/AutoJS Pro、引擎和目标 Android 版本，再查实际 API 行为。根因：平台差异和版本坑被忽略。
- 反例 2：无障碍服务断开仍查 selector。错法：findOne 一直超时后继续调选择器。对法：先检查服务启用、授权、系统回收和当前包名，再抓控件树。根因：权限状态不是脚本逻辑的一部分。
- 反例 3：findOne 无超时。错法：findOne() 永久阻塞。对法：设置明确超时、失败截图、控件树摘要、重试上限和人工接管。根因：等待条件没有退出边界。
- 反例 4：同名 text 误点。错法：text("确定").click() 直接点。对法：加 packageName、className、父子关系、boundsInside、clickable 和页面状态校验。根因：选择器唯一性不足。
- 反例 5：坐标未适配分辨率。错法：click(900,1800) 写死。对法：按 bounds/比例/DPI/状态栏/导航栏换算，并做区域截图校验。根因：坐标兜底没有多分辨率适配。
- 反例 6：swipe 后立刻取列表。错法：滑动后马上读节点。对法：等待列表稳定、目标文本出现、滚动结束或截图特征变化。根因：动画和异步加载竞态未处理。
- 反例 7：captureScreen 拿旧图。错法：截图权限弹窗后直接 OCR。对法：确认授权成功、等待新帧、记录截图时间戳和页面 Activity。根因：截图管线和页面状态不同步。
- 反例 8：OCR 当精确控件。错法：OCR 识别到文字就点固定坐标。对法：OCR 只辅助，结合区域、置信度、控件树、截图校验和失败兜底。根因：视觉识别误差被当成确定信号。
- 反例 9：WebView 空树盲改。错法：控件树为空就改成坐标狂点。对法：记录 WebView 版本、H5 路由、加载状态、截图特征和业务完成信号。根因：WebView 可访问性暴露受版本和页面实现影响。
- 反例 10：Intent 启动不校验结果。错法：startActivity 后直接执行下一步。对法：校验 package/activity、首屏控件、登录态和弹窗。根因：启动成功不等于页面可操作。
- 反例 11：保活靠死循环。错法：while true 高频截图/OCR 防掉线。对法：前台服务、通知、低频调度、电量优化说明、用户停止入口和资源释放。根因：把系统后台限制误当成可用 CPU 循环解决。
- 反例 12：Android 12-15 权限变化未测。错法：只在旧 Android 10 设备验证。对法：覆盖 Android 12、13、14、15 的通知、前台服务、后台启动、截图和媒体权限差异。根因：系统版本限制导致线上失败。
- 反例 13：日志无证据。错法：只输出“点击成功/失败”。对法：输出 step、selector、bounds、package/activity、duration、retry、screenshotId、device 和异常堆栈。根因：稳定性日志不可追溯。
- 反例 14：越界自动化。错法：要求批量点赞、刷评论、抢购、绕风控或账号养号。对法：拒绝滥用，转为合法测试、无障碍辅助或风控风险说明。根因：任务目标违反平台规则和安全边界。
- 反例 15：副作用动作盲重试。错法：提交表单后页面卡住，脚本继续点提交。对法：提交类动作必须有唯一键、结果回读、超时停止和人工确认。根因：不确定状态下重试会造成重复提交。
- 反例 16：离开目标 App 仍继续点。错法：跳到系统设置或支付确认页后脚本按原坐标继续点击。对法：每步校验 package/activity 和页面签名，越界立即停下。根因：自动化缺少运行边界。
- 反例 17：权限弹窗被当业务按钮。错法：固定点“允许”区域，误点系统弹窗或广告弹层。对法：区分系统权限、App 弹窗、业务按钮和遮罩层，逐类处理。根因：页面层级和权限流没建模。
- 反例 18：失败后没有可恢复状态。错法：脚本异常退出，下次从头跑导致重复处理已完成项。对法：记录 runId、item key、checkpoint、成功/跳过/失败/待人工确认状态。根因：批处理没有幂等和断点续跑。
- 反例 19：截图/OCR 泄露隐私。错法：把含手机号、地址、订单、头像的截图上传到第三方 OCR 或日志。对法：本地裁剪、区域脱敏、最小保留和访问控制；敏感场景不外传。根因：证据链没有隐私边界。
- 反例 20：风控误判为脚本不稳定。错法：账号频繁异常后继续调坐标和等待。对法：记录账号状态、频率、平台提示和授权范围，触及风控边界停止。根因：平台策略不是技术等待能解决的问题。

## 2024-2026 新坑速查

- Android 12：后台启动限制、精确闹钟、前台服务启动窗口、隐私指示器影响截图/录屏证据，老式保活策略容易失效。
- Android 13：通知运行时权限影响前台服务可见性和用户感知，媒体权限拆分后截图/文件保存路径要复核。
- Android 14：前台服务类型、后台限制、隐式 Intent、安全导出和厂商任务管理更严格，脚本打包后要复测权限链。
- Android 15：后台启动、敏感权限、通知/悬浮窗体验、边到边显示和系统安全策略继续收紧；不要承诺旧脚本原样稳定。
- AutoX.js 与 Auto.js 4.x：API、引擎、模块加载、打包、权限弹窗和社区版本差异需要逐项验证，不能按名称相似默认兼容。
- Compose / Flutter / WebView 页面：控件树可能不稳定或暴露不足，必须结合截图/OCR、业务状态和区域校验。
- 厂商 ROM：MIUI、ColorOS、OriginOS、EMUI/HarmonyOS、OneUI 对无障碍保活、悬浮窗层级、自启动、锁屏运行和后台弹窗差异明显。
- 多分辨率与大屏：折叠屏、平板、横屏、多窗口、虚拟导航栏、字体缩放会破坏绝对坐标和截图模板。
- 隐私与合规：截图、OCR、日志、录屏容易携带个人信息；交付证据必须脱敏并保留可复核 ID。
- 反滥用边界：平台对刷量、批量账号、异常点击、抢购、绕风控更敏感；这类请求必须拒绝或转为合规测试说明。

## 与相邻技能的边界

- Android 开发/android-development（andr）：负责 Android 原生 App、Manifest、Kotlin/Java、WorkManager、前台服务、权限实现和发布签名；本技能只处理 AutoJS 脚本与设备自动化排障。
- 移动安全/mobile-security（msec）：负责授权移动安全评估、权限风险、存储/TLS/WebView/Root/Jailbreak 防护验证；本技能不提供绕过检测、攻击或规避风控流程。
- 协议分析/protocol-analysis（prot）：负责授权抓包、pcap、TLS/HTTP/WebSocket/gRPC/时序与协议证据；本技能只在自动化现象层记录网络状态，不做协议重放或改写。
- 逆向工程总控/reverse-engineering（rev）：负责授权样本静态/动态分析、反编译、调用图、保护识别；本技能不逆向第三方 App、不绕过混淆/加固/检测。
- JavaScript/TypeScript 开发/javascript-typescript-development（jsts）：负责 JS/TS 语言、Node、包管理、异步、构建和类型问题；本技能只处理 AutoJS 运行环境中的脚本落地和 Android 设备行为。
- 可观测性/observability（obs）：负责线上服务 SLI/SLO、日志指标链路、告警和事故响应；本技能只要求端侧稳定性日志、截图和设备证据。
- UI 设计实现/ui-design（u）：负责视觉层级、布局、组件状态、响应式和可访问性设计；本技能只校验自动化脚本识别 UI 的证据，不替代产品 UI 设计。
- 测试验证/test-engineering（tst）：负责测试策略、场景矩阵、自动化测试和回归证据；本技能提供 AutoJS 设备执行证据，复杂回归需转测试工程收口。
- 代码审计/code-audit（aud）：负责改动后的需求对账、影响面、安全质量和证据收口；本技能更新或脚本修改完成后必须用审计口径复核。
