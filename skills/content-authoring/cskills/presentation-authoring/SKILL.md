# Presentation Authoring

Presentation Authoring 负责把业务内容交付成可讲、可看、可导出的演示稿。目标是让叙事、页面结构、图表、版式、母版和导出结果形成闭环，而不是把文字塞进幻灯片。

## 适用范围

- PPT、PPTX、slide deck、路演、融资材料、业务汇报、项目复盘、销售方案、培训课件和会议演示。
- 母版、主题、版式、封面、目录、章节页、图表页、数据页、案例页、结尾页。
- 叙事重构、页间逻辑、标题改写、图表表达、信息密度、视觉统一和演讲节奏。
- 导出 PDF、图片、演讲者备注、放映检查、字体嵌入和跨设备打开校验。

## 不适用范围

- 只写文案、不制作或修改演示稿文件。
- 普通 UI 页面、网页、APP 界面、设计系统或前端样式。
- 只读学习、项目上手、仅了解 PPT 概念，没有创建、修改、导出或校验动作。

## 铁律

1. 每页只能服务一个主结论；标题优先写结论，不写“背景介绍”式空标题。
2. 先定受众、场景、时长、讲述者、投影环境和决策目标，再定页数和视觉风格。
3. Deck 结构必须有叙事弧线：问题、洞察、方案、证据、计划、请求或结论。
4. 图表页必须先确认数据来源、口径、单位、时间范围和想表达的比较关系。
5. 母版、字体、颜色、间距、页码、图标和图表样式要统一，不能逐页手工漂移。
6. 路演和对外汇报不得编造客户、收入、增长、案例、合规承诺或融资数据。
7. 交付前必须打开或渲染检查：字体、图片、图表、动画、备注、导出和宽高比。

## 强制流程

1. 锁定场景：确认受众、目标、时长、页数、语气、品牌模板、是否需要备注和交付格式。
2. 抽取素材：区分事实、观点、待确认数据、引用、图片、图表和必须保留内容。
3. 建叙事：先列 deck storyline，再定每页 one-message、证据和页型。
4. 设计母版：统一尺寸、字体、色板、页眉页脚、页码、章节样式和图表样式。
5. 制作页面：优先使用图表、表格、流程、对比、时间线和案例结构；减少大段正文。
6. 校对口径：检查数字、单位、时间、客户名、术语、图例、脚注和页间一致性。
7. 渲染导出：打开 PPTX，导出 PDF 或图片，检查错位、溢出、缺字、糊图和动画依赖。
8. 交付说明：列出页数、结构、关键改动、待确认项和验证结果。

## 场景执行卡

## 渲染验收模板

- 记录 PPTX、PDF/图片导出、打开工具/版本、页数、宽高比、字体替换、文本溢出、图片缺失、图表错位、隐藏页和备注状态。
- PowerPoint、Keynote、LibreOffice Impress、Google Slides 至少按目标交付选择一个真实查看器；跨应用只报告已验证范围。
- 动画、视频、音频、嵌入字体、链接文件、演讲者备注、外部数据图表必须列可验证/不可验证状态。
- 交付包要说明 PPTX、PDF、图片导出、源数据/生成脚本、字体清单和未验证对象清单。

### 业务汇报

- 查：汇报对象、周期、核心指标、目标差距、原因、动作、风险和需要决策的事项。
- 做：用结论标题串起主线；指标页突出趋势、差距和归因，不堆表。
- 验：所有数字有口径，图表单位一致，行动项可落地。

### 路演 / 融资 Deck

- 查：市场、痛点、产品、商业模式、增长、客户、竞争、团队、财务和融资用途。
- 做：先讲机会和证据，再讲计划；夸张表述和未经证实数据标待确认。
- 验：叙事完整、页数可讲、敏感数据无泄露、导出无错位。

### 图表页

- 查：数据来源、分母、时间范围、筛选条件、异常值和对比关系。
- 做：选择适合比较、趋势、构成、分布或流程的图形；标题写结论。
- 验：轴、单位、图例、标注和脚注一致，色彩不误导。

## 复杂对象与低级错误

- 复杂图表要检查嵌入数据表、图表缓存、轴/单位/图例、外链数据、组合图/瀑布图/地图图降级和图片型图表不可编辑风险。
- 低级错误包括页码错、母版漂移、元素越界、字号过小、颜色对比不足、标题无结论、脚注缺口径和中文字体丢失。
- Google Slides / Keynote 任务必须补导入导出、字体替换、母版丢失、动画降级、评论权限和分享范围验证。
- PPTX 导出为 PDF 不等于放映通过；放映场景还要检查投影比例、备注、动画触发、外链媒体和演讲时长。

## 输出要求

- 输出 deck 结构、每页主结论、视觉/母版策略、数据待确认项和渲染验证结果。
- 未实际打开、渲染或导出时，必须写未验证。
- 对外材料必须标注未经确认的数字、客户、收入、法律和合规表述。

## 反例

- 只让写一段演讲稿或标题，不制作演示稿。
- 普通网页或产品 UI 页面。
- 只读学习 PPT 技巧，没有文件交付动作。
