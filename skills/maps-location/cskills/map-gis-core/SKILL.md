# 地图 GIS 核心开发

首次自称：地图 GIS 核心开发（map-gis-core，兼容 slug: map-gis-core）。

定位：把地图、定位、空间数据、路线、POI 和瓦片能力从“能显示”收敛为“坐标正确、平台一致、额度可控、隐私合规、性能可测、失败可降级、发布可回滚”。本技能综合官方文档原则与成熟工程实践做概念对齐，不复制厂商长文或示例代码。

## 定位/适用范围

- 适用于 Web 地图、后端/WebService GIS 服务、Android、iOS、小程序、BFF、地图数据管线、空间检索、路线规划、逆地址解析、POI 搜索、室内/室外定位、离线地图和地理可视化。
- 适用于接入或迁移高德、百度、腾讯、Google、Mapbox、OpenStreetMap、ArcGIS、Cesium、OpenLayers、Leaflet、MapLibre、GDAL/GEOS/JTS/JSTS、PostGIS/MySQL Spatial 等能力时的工程门禁。
- 覆盖坐标系 WGS84、GCJ-02、BD-09、CGCS2000、EPSG:3857/Web Mercator、地方投影、企业自有网格与屏幕像素坐标的转换、存储、展示和接口契约。
- 覆盖瓦片、矢量、栅格、GeoJSON、MVT、WMS、WMTS、WFS、KML、GPX、Shapefile/GeoPackage 导入导出及其安全与性能边界。
- 覆盖地理编码/逆地理编码、路线规划、导航、路径吸附、地理围栏、POI、行政区划、轨迹回放、热力图、聚合、测距测面、空间关系判断。
- 不替代厂商官方资质、测绘法规、地图审图、隐私法务、采购计费合同；涉及合规必须找对应责任人确认。
- 触发本技能：需求包含地图 SDK、GIS、坐标/投影、定位、地理编码、路线/导航、POI、瓦片、矢量/栅格、GeoJSON/MVT/OGC 服务、空间数据库/索引、轨迹、地理围栏、离线地图、地图发布或地图安全隐私。
- 不触发本技能：source map、hash map、roadmap、mind map、site map、map/filter/reduce、普通“映射”数据结构、旅行攻略/路线建议/常识问答、只读学习咨询、入门教程、README/依赖清单里仅出现地图库且没有修改/调试/测试/部署动作、仅闲聊提及高德/百度/腾讯/Google/Mapbox 等 provider 名称而没有接入/迁移/排障/发布意图。
- 仅当用户要求真实开发、排障、测试、部署、审计、迁移或发布地图/GIS能力时执行本技能；没有代码或配置变更意图时，只给概念边界并停止。

## 铁律

- 坐标系不明禁止入库、禁止缓存、禁止接口透传、禁止上线；每个字段、文件、表、Topic、API、埋点都要标明坐标系、精度、单位、来源和时间。
- 中国大陆公开地图展示、定位纠偏、底图叠加必须按所用厂商和法规要求处理 GCJ-02/BD-09/CGCS2000 等坐标差异，不得把 WGS84 直接叠到国内加密底图上。
- EPSG:3857 是显示投影，不等于真实经纬度；面积、距离、缓冲、相交、最近点等严肃空间计算不得默认在 Web Mercator 上直接算。
- 经纬度顺序必须统一：接口、数据库、GeoJSON、SDK、日志中明确 lon/lat 或 lat/lng；GeoJSON 坐标顺序按经度、纬度处理，UI SDK 参数另行核对。
- 密钥不得写入客户端可逆源码、仓库、日志、前端环境包或小程序包；前端密钥必须配域名/包名/bundle id/SHA-1/IP 白名单、限额、最小权限和监控告警。
- 定位、轨迹、住址、常去地点、精确 POI 查询属于敏感或准敏感数据；采集前要最小化、告知授权、可撤回、分级脱敏、限定保留期。
- 第三方地图 API 调用必须有配额、账单、限流、缓存、降级和应急开关；不得把无限自动重试接到付费 API。
- 路线、地理编码、POI、行政区划结果不能只按一次厂商返回写死业务真相；必须记录 provider、版本、时间、置信度和可重算策略。
- 离线包、瓦片缓存、矢量数据、POI 数据不得绕过授权范围分发；商用和离线使用必须核对 license、审图和服务条款。
- 发布必须附坐标样本、跨平台截图/录屏、网络失败表现、配额账单保护、隐私权限文案和回滚开关证据。

## 快速总则

- 先定业务事实坐标系，再定展示坐标系；存储层推荐保留原始坐标、标准化坐标、坐标系枚举、采集精度、来源和转换链路。
- 地图是多端系统：Web、后端、Android、iOS、小程序对坐标顺序、权限弹窗、生命周期、后台定位、WebView、小程序域名和 SDK 版本都有差异。
- 底图、业务图层、定位点、搜索结果、路线线形、行政区边界必须逐层核对坐标系、投影、缩放级别、裁剪范围、zIndex、透明度和更新频率。
- 瓦片类能力先确认协议：XYZ/TMS、WMTS、WMS、WFS、MVT、栅格/矢量、坐标参考系、tile matrix set、style、缓存头、鉴权方式和跨域策略。
- 空间数据先做边界治理：几何合法性、闭合环方向、MultiPolygon、洞、多部件、跨 180° 经线、极区、空几何、超大几何、简化精度和拓扑错误。
- 地理编码/POI/路线先确认业务语义：城市限定、语言、行政区、坐标返回系、模糊匹配、置信度、分页、排序、驾车/步行/骑行/公交策略和失败兜底。
- 客户端默认做节流、防抖、视口裁剪、聚合、按需加载、低端机降级；服务端默认做缓存、批量、异步、限流、熔断、观测和重试预算。
- 跨厂商迁移必须建立对照矩阵：坐标系、API 返回字段、错误码、限额、计费、授权范围、风格样式、POI 分类、路线策略、定位权限和合规条款。
- 一切地图问题都要能复现：给出坐标、坐标系、平台、SDK 版本、地图 provider、缩放级别、时间、网络、设备、权限状态和截图。

## 核心速查卡

- CRS/EPSG：每个空间字段和数据文件都要写 SRID/EPSG、轴顺序、单位、适用区域、误差来源和转换链路；缺 CRS 的 Shapefile、CSV、GeoJSON、WKT/WKB 只能进入隔离区，不能直接入业务表。
- WGS84/GCJ-02/BD-09/CGCS2000：先确认数据采集地、展示底图和 provider；境外常见 WGS84，国内互联网底图常见 GCJ-02 或 BD-09，政府/测绘数据可能是 CGCS2000 或地方投影，禁止凭字段名猜。
- Web Mercator：EPSG:3857/900913 适合瓦片切图和屏幕展示；距离、面积、buffer、intersection、nearest 等计算先转合适投影或使用大地测量算法，并记录单位。
- lon/lat vs lat/lng：GeoJSON、PostGIS ST_MakePoint 常按 lon,lat；部分移动 SDK、表单和三方 API 可能叫 latitude/longitude 或 lat/lng；接口必须用字段名和示例值防反转。
- GeoJSON：默认坐标数组是经度、纬度，CRS 成员在新版规范中不作为主流契约；跨系统交换时用外层字段、接口文档或文件名声明 CRS，并限制属性白名单。
- WKT/WKB/EWKT/EWKB：WKT 便于排障，WKB 便于二进制传输，EWKT/EWKB 可携带 SRID；解析前校验几何类型、SRID、闭合环、空几何和坐标范围。
- MVT：矢量瓦片是 tile-local 坐标和属性裁剪后的传输格式，不等于原始 GIS 数据；必须锁定 extent、buffer、layer 名、字段白名单、简化策略和 style 版本。
- Tile schema：确认 XYZ 或 TMS、z/x/y 原点、zoom 范围、tile size、DPR/Retina、bbox 计算、matrix set、缓存 key、过期策略、版权标注和 fallback。
- 地理编码/逆地理编码：输入必须清洗、限定城市/国家、记录 provider/语言/置信度/返回坐标系；低置信度不能自动覆盖用户确认地址。
- POI：分类体系、营业状态、地址层级、电话、坐标、行政区和别名会随 provider 变化；缓存 TTL、授权范围和敏感查询脱敏要写进契约。
- Route/distance matrix：路线规划返回路径，距离矩阵返回多起终点成本；都要记录出行方式、路况时间、避让策略、途经点上限、计费模型和失效时间。
- Spatial index：点/线/面检索优先 bbox + GiST/SP-GiST/R-tree/quadkey/geohash/H3/S2 等索引预过滤，再做精确空间谓词；索引坐标系和查询半径单位必须一致。
- Buffer/intersection：缓冲、相交、包含、覆盖、最近点等操作要先修复无效几何和选择计算坐标系；输出面积、长度和半径必须附单位和误差预算。
- Projection transform：转换函数必须声明源 CRS、目标 CRS、使用库/算法、适用区域和是否可逆；禁止链式重复纠偏，禁止把屏幕像素坐标当经纬度。
- Privacy/location：精确位置、轨迹、家庭/工作地推断和常去 POI 属于高风险数据；默认最小采集、分级模糊化、短保留期、访问审计和用户撤回。
- License/attribution：底图、瓦片、POI、行政边界、路网和离线包分别核对 license、商用授权、缓存授权、再分发限制、审图要求和版权标注展示。

## 强制流程

1. 需求定界：明确业务目标、用户位置敏感级别、覆盖平台、国家/地区、地图 provider、底图来源、空间数据来源、精度要求、离线要求、计费预算和回滚要求。
2. 坐标账本：列出输入、存储、服务计算、接口输出、客户端展示、日志埋点、第三方调用的坐标系、单位、经纬顺序、精度、转换函数和误差预算。
3. 数据契约：为 GeoJSON/WKT/WKB/MVT/WMS/WMTS/WFS/瓦片/路线/距离矩阵/POI/定位接口定义字段、CRS、bbox、zoom、时间、provider、版本、缓存、错误码和兼容策略。
4. 平台矩阵：分别核对 Web、后端/WebService、Android、iOS、小程序的 SDK 版本、权限、密钥限制、网络域名、生命周期、后台能力、包体和降级策略。
5. 安全隐私评估：确认权限文案、最小采集、脱敏粒度、保留期、访问控制、审计、跨境/第三方传输、密钥保护、配额告警和账单上限。
6. 性能预算：设定首屏地图时间、瓦片请求数、图层数量、点线面数量、轨迹点数、内存、CPU、帧率、电量、服务 QPS、缓存命中率和批量上限。
7. 实现前样本：准备标准坐标点、国内外样本、边界样本、失败样本、低端设备样本、无权限样本、无网络样本、超限样本和截图基线。
8. 开发实现：按平台最小权限接入；按视口/zoom 加载数据；按 provider 错误码映射；所有外部 API 调用接入限流、熔断、缓存和可观测。
9. 攻击面检查：地图代理、瓦片转发、文件上传、批量查询、回调 URL、CORS、Referer、签名、缓存键、日志和导出下载逐项做滥用与越权检查。
10. 验证门禁：执行坐标、图层、定位、POI、路线、瓦片、安全、隐私、配额、性能、离线、兼容、回滚测试；证据不足不得发布。
11. 发布闭环：灰度、监控、账单告警、错误分桶、用户反馈入口、远程开关、回滚方案、厂商状态页和证据归档齐备后上线。
12. 停止条件：缺坐标账本、缺 license/版权/审图确认、缺隐私授权、缺 key 限制、缺配额保护、缺回滚开关、缺真机/真实环境证据时停止上线并列阻塞项。

## 场景执行卡

### 1. Web 地图与前端可视化

- 证据：浏览器矩阵、地图 SDK/渲染库、底图 provider、坐标系、CSP、Referer 限制、CDN、图层数量、最大点线面规模。
- 动作：地图容器 resize、按 zoom/viewport 加载、marker/label 聚合、canvas/WebGL 降级、瓦片缓存头、跨域、token 刷新和错误态要处理。
- 验证：首屏、缩放拖拽、海量点、低端设备、移动浏览器、暗色模式、离线/弱网、密钥被拒、CORS、provider 超限、429/403/5xx、截图基线和回滚开关。

### 2. 后端/WebService GIS

- 证据：数据库空间索引、SRID、API 契约、QPS、批量大小、缓存、第三方 API 预算、异步任务和审计需求。
- 动作：入库校验 SRID 和几何合法性；空间查询用索引和 bbox 预过滤；buffer/intersection/nearest 先选计算坐标系；外部调用批量化、缓存化、限流化；长任务异步化。
- 验证：SRID 错误、经纬反转、无效几何、跨租户数据、超大 polygon、空间索引失效、半径单位错误、分页、并发、重试、熔断、429/403/5xx、账单上限、冷启动和回滚。

### 3. Android 地图/定位

- 证据：target SDK、定位权限、后台定位、厂商 SDK、包名/SHA-1 key 限制、电量预算、国内外地图策略。
- 动作：运行时权限、精确/大致位置、前后台切换、定位间隔、传感器融合、生命周期释放、混淆规则、离线包和低电量策略必须明确。
- 验证：拒绝权限、仅大致位置、后台限制、冷启动定位、网络定位失败、GPS 弱、低端机帧率、key 不匹配、渠道包签名差异。

### 4. iOS 地图/定位

- 证据：Info.plist 权限文案、Always/WhenInUse 策略、后台模式、bundle id key 限制、iOS 版本、隐私清单和电量预算。
- 动作：权限请求时机、精确位置开关、临时精确位置、后台定位蓝条/指示、定位频率、SDK 生命周期和内存释放要按平台规范实现。
- 验证：拒绝权限、仅大致位置、临时精确失败、后台被系统挂起、低电量、无网络、key 限制、App Store 隐私说明和崩溃日志。

### 5. 小程序地图能力

- 证据：小程序平台、合法域名、地图组件限制、插件/SDK、用户授权、隐私弹窗、包体、分包和审核要求。
- 动作：经纬顺序、GCJ-02 要求、组件层级、cover-view 限制、授权失败、地图上下文生命周期、云函数代理和域名白名单要逐项核对。
- 验证：真机、开发者工具差异、拒绝授权、二次授权、弱网、合法域名失败、包体超限、审核隐私条款、平台 API 兼容。

### 6. 坐标转换与投影

- 证据：源坐标系、目标坐标系、使用地区、转换算法来源、误差容忍、是否官方/厂商要求、是否批量和是否可逆。
- 动作：转换链路只做必要步骤；禁止重复纠偏；转换函数带单元测试；边界外坐标不盲转；面积距离使用适合投影或大地测量算法。
- 验证：标准点、国内外点、边界点、经纬反转、重复转换、浮点精度、批量性能、日志采样、与厂商点位截图比对。

### 7. 瓦片、矢量、栅格与 OGC 服务

- 证据：XYZ/TMS/WMTS/WMS/WFS/MVT 协议、CRS、tile matrix、zoom 范围、style、缓存、鉴权、版权和更新频率。
- 动作：tile URL 参数白名单；XYZ/TMS 原点、z/x/y、tile size、DPR、bbox/zoom 限制；WMS/WFS 防超大查询；MVT extent/buffer/layer/style 锁定；缓存分层；版权标注不可隐藏；地图代理禁止任意 URL 转发和内网探测。
- 验证：不同 zoom、边界 tile、Retina、透明背景、XYZ/TMS 翻转、样式版本、缓存失效、服务 4xx/5xx、跨域、超大 bbox、license/attribution 展示、SSRF 拒绝和 provider fallback。

### 8. GeoJSON/WKT/WKB/MVT/空间文件导入导出

- 证据：数据量、坐标系、属性字段、编码、几何类型、拓扑质量、敏感字段、导出授权和目标系统要求。
- 动作：导入前校验 schema、大小、MIME/扩展名、解压比例、几何合法性、SRID/EPSG、轴顺序、属性白名单；导出前脱敏、简化、裁剪、限量、异步和审计。
- 验证：空几何、坏环、洞、多部件、中文编码、超大文件、压缩炸弹、恶意属性、坐标越界、WKT/WKB SRID 缺失、导出权限、下载过期和重试。

### 9. 地理编码、逆地理编码、POI

- 证据：provider、城市/国家范围、语言、POI 分类、分页、排序、置信度、计费、缓存许可和用户隐私等级。
- 动作：输入清洗、城市限定、模糊匹配标记、结果置信度、provider 版本、失败兜底、缓存 TTL、批量限流和审计必须实现。
- 验证：重名地址、跨城市、无结果、低置信度、多语言、分页末页、provider 超限、缓存命中、敏感查询脱敏和账单异常。

### 10. 路线规划、距离矩阵、导航与轨迹

- 证据：出行方式、实时路况、避让策略、途经点上限、坐标系、路线时效、导航授权、轨迹采样频率和纠偏策略。
- 动作：路线和距离矩阵结果保存 provider、策略、时间、坐标系和失效时间；轨迹做采样、压缩、异常点过滤、隐私脱敏；导航失败有备用方案。
- 验证：多起终点矩阵、途经点上限、跨城、无路、禁行、坐标漂移、轨迹断点、路径吸附错误、实时路况失效、费用超限和旧路线重算。

### 11. 离线、弱网与灾备

- 证据：离线区域、数据授权、包大小、更新频率、缓存策略、失效时间、低存储和灾备 provider。
- 动作：离线包签名、版本、增量、清理、失败回滚；弱网显示骨架/旧缓存；provider 故障切换必须可远程开关。
- 验证：无网启动、缓存过期、包损坏、磁盘满、增量失败、provider 429/403/5xx、DNS 失败、低速网络、fallback provider、离线版权和回滚。

### 12. 密钥、配额、账单与发布运营

- 证据：key 类型、限制条件、服务权限、QPS、日限额、计费单价、告警阈值、负责人、应急降级和合同授权。
- 动作：按环境拆 key；最小权限；服务端代理敏感调用；请求签名；日志脱敏；预算告警；异常流量自动限流；灰度 key 可撤销。
- 验证：key 泄露演练、Referer/IP/包名限制、超额返回、账单突增、限流策略、告警到达、降级文案和禁用开关。

## 验证门禁

- 坐标门禁：固定金标样本（如天安门、上海陆家嘴、香港、台北、纽约等已知 WGS84/GCJ-02/BD-09 多系参考点）、国内外点、边界点、不同坐标系点、经纬反转点、重复转换点、距离/面积样本全部通过；与厂商 SDK/官方工具截图比对误差在允许范围内。
- 图层门禁：底图、业务图层、定位点、POI、路线、行政区、栅格/矢量在目标 zoom 和平台上对齐，无明显偏移和 zIndex 错乱。
- 平台门禁：Web、后端/WebService、Android、iOS、小程序各有至少一组真机或真实环境证据；不能只用模拟器或开发者工具替代。
- 权限门禁：拒绝定位、仅大致位置、后台定位限制、二次授权、隐私弹窗、撤回授权、无权限 API 分支都被验证。
- 安全门禁：密钥限制、服务端签名、接口鉴权、对象级授权、日志脱敏、CORS/CSP、域名白名单、上传文件校验、SSRF 防护全部通过。
- 配额门禁：限流、缓存、熔断、重试预算、账单告警、超额降级、429/403/5xx、provider 故障、fallback provider、远程开关全部演练。
- 性能门禁：首屏、帧率、内存、CPU、电量、瓦片请求数、空间查询耗时、批量任务耗时、缓存命中率满足预算。
- 数据门禁：几何合法性、SRID、空间索引、bbox 裁剪、分页、导入导出大小、敏感字段、license 和版权标注通过。
- 计算门禁：距离、面积、buffer、intersection、contains、nearest、distance matrix 在正确 CRS/单位下验证；Web Mercator 仅用于展示的场景必须写明。
- 发布门禁：灰度计划、监控面板、告警人、回滚开关、厂商状态页、版本号、截图/录屏、样本坐标和测试报告齐备。

## 安全边界

- 不提供绕过地图授权、破解离线包、隐藏版权标注、批量爬取 POI/瓦片、规避计费、绕过审图、绕过跨境/位置数据限制或违反服务条款的方案。
- 不建议在无用户授权下采集精确位置、后台连续定位、推断住址/工作地或跨应用共享轨迹；不得用“匿名”掩盖可重识别轨迹。
- 不把第三方 key、签名 secret、服务端 token、个人位置样本、轨迹原始数据写入前端包、公开日志或测试报告。
- 不把地图代理做成开放转发器；必须拒绝内网地址、云元数据地址、file 协议、重定向绕过、超大响应和非白名单 provider。
- 不为 bulk scraping、撞库式 POI 查询、瓦片批量下载、离线包再分发、版权标注移除、额度规避或风控绕过提供实现步骤。
- 不将地图结果作为人身安全、救援、执法、金融风控等高风险唯一依据；高风险场景必须增加人工复核、冗余数据源和合规审批。
- 不在缺少 license、版权、审图、隐私、跨境传输确认时发布地图数据、离线包、行政边界或 POI 数据。

## 反例库

- 反例：数据库只存 lat、lng，没有 SRID 和来源。后果：迁移 provider 或叠加底图时整体偏移且无法追溯。
- 反例：把 WGS84 轨迹直接画在国内 GCJ-02 底图。后果：车辆、门店、路线偏移，用户认为定位错误。
- 反例：前端 key 无域名限制，服务端 key 打进 App 包。后果：盗刷账单、额度耗尽、敏感接口被滥用。
- 反例：路线规划失败时客户端无限重试。后果：付费调用暴涨，用户仍无结果，服务被限流。
- 反例：只在开发者工具验证小程序地图。后果：真机组件层级、授权、合法域名和性能问题上线才暴露。
- 反例：用 EPSG:3857 坐标直接计算面积。后果：面积误差随纬度放大，报表和结算错误。
- 反例：GeoJSON 上传不限制大小和属性。后果：内存爆、脚本注入、敏感字段泄露、空间查询拖垮数据库。
- 反例：地图上线无账单告警。后果：key 被盗或循环请求时直到出账才发现。
- 反例：POI 结果永久缓存且不记录 provider 版本。后果：过期地址无法解释，合规与授权边界不清。
- 反例：后台持续定位没有可见说明和停止机制。后果：隐私违规、耗电投诉、应用审核失败。
- 反例：GeoJSON 坐标按 lat,lng 解析。后果：点位落到错误国家或海上，排障时日志还看似“有坐标”。
- 反例：MVT 当作原始数据源回写业务库。后果：简化和裁剪后的几何丢失精度，属性也可能被截断。
- 反例：距离矩阵没有记录路况时间和策略。后果：次日重算结果不同，调度和费用争议无法复现。
- 反例：空间索引建在 WGS84 点上却用米作为半径直接查询。后果：筛选范围随纬度失真，近点漏召或远点误召。
- 反例：瓦片 URL 只拼 z/x/y，不区分 XYZ/TMS。后果：部分 zoom 下南北翻转，边界 tile 错位。
- 反例：导出行政边界时隐藏 attribution。后果：违反授权条款，商用发布被投诉或下架。

## 相邻技能边界

- source-map/构建调试：source map、bundle 映射、栈追踪还原属于构建或调试技能；除非问题涉及真实地图坐标/图层/瓦片，否则不触发本技能。
- 数据结构/算法：hash map、map/filter/reduce、普通字段映射、配置映射属于语言或架构技能；没有 GIS 坐标、空间数据或地图 SDK 时不触发本技能。
- 产品/内容规划：roadmap、mind map、site map、旅行线路建议、景点攻略、只读学习和 onboarding 由产品、内容或学习技能处理；没有开发、调试、测试、发布动作时本技能只给边界提醒。
- 依赖证据：README、package、Gradle、Podfile 里仅出现地图库依赖但没有地图功能变更，不足以单独触发本技能；需要结合代码、配置、接口或发布动作判断。
- JavaScript / TypeScript 开发 / javascript-typescript-development（jsts）：负责 JavaScript 空间几何计算、拓扑关系、缓冲、相交、简化等库级问题；本技能负责地图 GIS 端到端坐标、平台、数据、合规和发布门禁。
- Vue 开发 / vue-development（vue）：负责 Vue 组件、状态、渲染和前端工程；地图组件接入 Vue 时由本技能定义 GIS 门禁，Vue 技能处理框架实现质量。
- API 工程 / api-engineering（api）：负责 API 契约、鉴权、错误模型和版本兼容；地图服务接口需同时满足本技能坐标/空间语义和 api 技能契约要求。
- 测试验证 / test-engineering（tst）：负责测试策略、自动化、覆盖率和测试工程；本技能提供地图 GIS 必测样本、平台矩阵和发布证据清单。
- 代码审计 / code-audit（aud）：负责代码审计和通用质量审查；地图数据泄露、坐标错误和计费风险由本技能给出领域审计点。
- Web 安全 / web-security（wsec）：负责 Web 安全；地图前端 key、CORS/CSP、上传 GeoJSON、XSS 和接口滥用需与本技能联合使用。
- Apple 全链路开发与发布 / apple-development（appl）：负责 Apple 平台工程；iOS 地图定位、权限、隐私清单和 App Store 审核需与本技能联合使用。
- Android 开发 / android-development（andr）：负责 Android 平台工程；Android 定位权限、后台限制、签名 key 和性能需与本技能联合使用。
- Kotlin 开发 / kotlin-development（kt）：负责 Kotlin 语言和 Android/KMP 实现细节；地图业务的坐标和 SDK 领域规则仍由本技能定义。
- Flutter 开发 / flutter-development（fltr）：负责 Flutter 跨端工程；Flutter 地图插件、平台通道和性能需叠加本技能的 GIS 门禁。
- 微信小程序 / wechat-miniprogram（wcmp）：负责小程序工程；小程序地图组件、授权、合法域名、审核和包体需叠加本技能的 GIS 门禁。
- 数据库工程 / database-engineering（db）：负责数据库设计、索引、迁移和数据一致性；空间表、SRID、空间索引、几何合法性需与本技能联合使用。
- 性能工程 / perf-engineering（pfe）：负责前端性能；海量点、瓦片、WebGL、首屏和低端机性能需与本技能联合使用。
- 发布部署 / release-engineering（rls）：负责发布、灰度、回滚和变更管理；地图 provider、key、账单和离线包发布需满足本技能证据。
- 可观测性 / observability（obs）：负责可观测性；地图错误分桶、provider SLA、账单告警、定位失败率、瓦片失败率和空间查询耗时需与本技能联合使用。
