## 约束与非目标

- 约束：未知命令、外部服务、账号、密钥和部署环境必须标为待确认，不得伪造。
- 约束：完成并验证修改后进入原子 commit 闭环；每个 commit 只包含一个完整工作包；不默认 push、merge 或 PR。
- 约束：空目录、新想法或需求不清晰时，先进入需求访谈；未经确认的需求、角色、权限、数据来源、商业化和合规判断必须标 `待确认`。
- 非目标：在项目目标、验收标准和验证方式明确前，不扩大到多功能实现。
