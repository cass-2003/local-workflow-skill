## J-SOP 代码审查

### 上下文恢复
1. `git diff --stat HEAD` 查看变更范围
2. `npx tsc --noEmit` 编译状态

### 审查维度

1. **类型安全**：新增/修改的类型是否正确，`any` 使用是否合理
2. **消息一致性**：发送方和接收方的消息结构是否匹配
3. **DOM 安全**：innerHTML 赋值是否有 XSS 风险（外部数据需 escapeHtml）
4. **错误处理**：新代码是否有 try-catch，失败路径是否合理
5. **selector 稳定性**：新选择器是否有兜底，是否过于宽泛
6. **命名规范**：变量/函数名是否自解释，是否与现有风格一致
7. **注释完整**：修复标注 ID、JSDoc、复杂逻辑说明
8. **性能**：是否有不必要的循环/重复 DOM 查询/内存泄漏
9. **向后兼容**：types.ts 改动是否影响现有消费者

### 输出格式

```markdown
## Code Review: {日期}

### 变更概览
- 文件数：N
- 新增/删除行：+N/-N

### 发现
| # | 级别 | 文件:行号 | 问题 | 建议 |

### 总结
- 通过/需修改/阻塞
```
