# 抖音小程序

抖音小程序（douyin-miniprogram，兼容 slug: dymp）负责本技能描述范围内的定位、执行、验证和交接边界；旧短 slug 仅作兼容 alias/URL 主键，不作为规范技能名。

定位：只负责抖音小程序原生端实现与排障证据，不替代通用 JS、后端 API、UI 视觉、测试工程或发布工程。任何结论先落到端能力、页面入口、平台审核、证据四要素，再决定是否联动相邻技能。

## 快速总则：端能力 / 页面入口 / 平台审核 / 证据

1. 端能力先实查：确认开发者工具、基础库、宿主 App、iOS/Android、tt.canIUse、后台能力开通与权限声明；为支付、直播、短视频、授权等列宿主/基础库矩阵，不确定能力不可凭微信或 H5 经验迁移。
2. 页面入口先枚举：app.json pages、tabBar、subPackages、preloadRule、分享、扫码、短视频挂载、直播小雪花、POI、web-view 回跳、Push、二维码都算入口。
3. 平台审核先前置：登录、支付、隐私、手机号、位置、相机、相册、麦克风、剪贴板、订阅/消息、直播/视频/POI 挂载、虚拟商品和交易组件必须有用户动作、用途说明、拒权降级、再次授权入口、测试账号和审核备注；页面加载即弹授权、拒权白屏、无替代路径均判高风险。
4. 证据先落地：结论绑定 ttml/ttss/js/json 文件、project.config.json、基础库、宿主版本、真机/体验版/线上环境、日志、截图、录屏或接口响应。
5. 原生边界：页面结构用 ttml，样式用 ttss，逻辑用 Page/Component，配置用 app.json/page json/project.config.json，能力用 tt.*；跨端生成物要审 Taro、uni-app、Remax 的抖音目标配置，发现 wx.*、my.*、Web DOM 结论要先拦截。
6. 数据更新：setData 只传必要路径，滚动、视频、直播、交易页禁止大对象高频更新；性能结论必须有性能面板或真机录屏证据。
7. 登录链路：tt.login 的 code/anonymousCode 只作为换取会话材料，jscode2session 要同时处理 anonymous_code、匿名态迁移、openid/unionid 绑定、token 续期和取消授权，必须服务端闭环。
8. 支付链路：tt.pay、pay-button、担保交易、交易组件的前端回调只做用户提示，发货/履约/退款/结算以后端订单、异步通知验签、查单、退款单和幂等为准。
9. 发布链路：开发版、真机调试、体验版、审核通过、灰度发布、全量上线是不同状态；不得互相替代。

## 单技能工程门禁

1. 开发前先写抖音端闭环表：入口、页面、ttml/ttss/js/json、tt API、后端接口、缓存 key、权限、审核材料、真机/体验版验证点；表里缺一项就不能直接开写高风险链路。
2. 改页面必须同时核 app.json/page json、入口 query、onLoad/onShow、空态/加载/失败/拒权、弱网、返回栈和分享/挂载入口；只改 ttml 或 js 不算闭环。
3. 改接口必须同时核 request 域名、HTTPS、header、token、错误码、字段兼容、旧缓存、弱网重试、幂等、trace id 和服务端日志；API response 不能直灌 UI，必须做字段默认值和旧版本兼容。
4. 改登录必须同时核匿名态、登录态、openid/unionid、业务 userId、token 续期、取消授权、切换账号、缓存清理和隐私弹窗；客户端不得生成或信任业务身份。
5. 改支付/担保交易必须同时核预下单、金额来源、订单状态机、异步通知验签、查单、退款、结算、重复回调、取消/超时/未知态和 iOS/商品规则；客户端只展示，不裁定履约。
6. 改开放能力必须同时核用户动作、用途说明、拒权降级、再次授权入口、隐私协议、审核备注、低版本降级和多宿主矩阵；拒权白屏就是未完成。
7. 改短视频/直播/POI/分享必须同时核真实挂载入口、scene、query、半屏高度、返回路径、落地页相关性、能力开通、审核截图和不支持宿主降级；普通页面跳转成功不能代替挂载成功。
8. 改缓存/本地存储必须同时核用户维度、版本、过期、清理、退出登录、切换账号和授权取消；token、openid、手机号、支付参数、精确位置、后台 secret 不得明文长期落盘。
9. 改上传/媒体/UGC 必须同时核格式、大小、时长、临时文件、进度、取消、失败重试、服务端内容安全、审核中/驳回/申诉 UI 和投诉入口；本地预览不是公开展示证据。
10. 收口前必须有最小验收矩阵：开发者工具、iOS 真机、Android 真机、体验版、弱网、拒权、旧缓存、挂载/分享入口、支付取消/失败/未知态；无法验证要明说缺口。

## 硬禁止与低级错拦截

- 禁止把 app secret、支付密钥、商户私钥、服务端 token、后台接口签名密钥写进前端代码、配置、日志、截图或审核备注。
- 禁止用客户端 openid、userInfo、手机号、订单金额、支付 success、分享 query、web-view 回跳参数作为服务端最终事实。
- 禁止只测开发者工具、只测一个普通入口、只测 happy path、只测已授权账号，就说抖音小程序已完成。
- 禁止把微信/支付宝/浏览器 API 经验直接套到 tt.*；出现 wx.*、my.*、document、window、localStorage、DOM 事件时先拦截。
- 禁止忽略授权拒绝、接口超时、弱网、缓存脏数据、旧版本字段缺失、重复点击、重复回调、重复提交和切换账号。
- 禁止在 onShow 里无条件重拉并覆盖用户输入；禁止 setData 整页对象、高频列表大包和离页后继续 setData。
- 禁止支付、退款、发货、履约、结算、内容公开展示只靠前端状态推进；这些必须以后端可追溯状态为准。
- 禁止提审前才补隐私协议、权限用途、测试账号、挂载入口、支付路径、审核备注和客服复现信息。
- 禁止体验版通过后直接包装成审核通过或线上稳定；审核、灰度、全量、回滚各自要有证据。

## 场景执行卡

### 1. 工程配置 / 基础库 / 调试器

- 适用：开发者工具正常、真机异常、API 不存在、构建或预览差异。
- 先查：project.config.json、app.json、基础库、开发者工具版本、npm 构建、SJS/ES 支持、插件、第三方 SDK、能力组件和分包配置；跨端生成物查 Taro @tarojs/plugin-platform-tt 与 project.tt.json、uni-app mp-toutiao 与 pages.json、Remax @remax/toutiao。
- 动作：同一入口分别跑开发者工具、真机调试、体验版；用 tt.canIUse 做能力分支；记录最低可用基础库；插件/SDK/交易/直播/地图/客服/统计组件要锁版本、核授权范围、隐私采集、包体影响、宿主兼容和审核材料；合法域名校验关闭仅本地，体验版和线上必须按后台域名与 HTTPS 证书验证。
- 证据：工具版本、基础库、宿主版本、设备型号、报错日志、复现步骤、tt.canIUse 结果、跨端编译产物路径和差异摘要。
- 兜底：只有调试器通过时结论为部分覆盖或无法验证。

### 2. 页面入口 / 路由 / 生命周期

- 适用：新增页面、跳转白屏、onLoad/onShow 重复请求、返回栈状态错乱。
- 先查：app.json pages、tabBar、subPackages、page json、query 来源、冷启动/热启动、半屏返回。
- 动作：入口统一列清；onLoad 做一次性初始化，onShow 只做可重入刷新；异步请求带版本号或取消逻辑。
- 证据：页面路径、query 样本、生命周期日志、返回栈、入口录屏。
- 兜底：普通入口通过不代表挂载、分享、POI、直播入口通过。

### 3. ttml / ttss / 原生组件层级

- 适用：样式错位、点击穿透、安全区、暗黑模式、半屏、视频/地图/直播覆盖层。
- 先查：ttml 层级、ttss 选择器、rpx/px、安全区、fixed/sticky、cover-view、原生组件限制。
- 动作：降低节点复杂度；视频/地图/直播上方只用平台允许覆盖方案；半屏和全屏分别适配；input/textarea、键盘遮挡、focus、滚动定位、placeholder、错误提示、字体缩放、暗黑模式、色彩对比要真机验证。
- 证据：机型截图、视口尺寸、宿主、页面模式、关键节点样式、表单输入录屏。
- 兜底：只测单机型或调试器截图不算完成。

### 4. Component / 状态 / setData 性能

- 适用：组件复用、状态不同步、列表卡顿、白屏、滚动掉帧。
- 先查：usingComponents、props、triggerEvent、attached/ready/detached、setData 大小与频率、缓存恢复。
- 动作：组件注册/清理成对；路径式 setData；分页、懒加载、虚拟列表；视频/直播上下文离开时释放。
- 证据：性能面板、首屏耗时、setData 数据量、滚动帧率、卸载日志。
- 兜底：无性能基线只能写已优化待验证。

### 5. 分包 / 包体 / 资源边界

- 适用：包体超限、分包白屏、独立分包、预加载失效、弱网首进慢。
- 先查：app.json pages、subPackages、preloadRule、公共依赖、图片/字体/视频资源、分包内相对路径、组件/插件体积和目标宿主组件兼容矩阵。
- 动作：主包保留启动必要资源；分包资源就近；独立分包不引用主包私有逻辑；弱网提供加载与失败态。
- 证据：包体报告、主包/分包体积、组件兼容矩阵、首进耗时、弱网日志。
- 兜底：开发版能跳不代表线上包加载成功。

### 6. 登录 / 授权 / code2session

- 适用：tt.login、anonymousCode、jscode2session anonymous_code、手机号、用户资料、openid 绑定、会话过期。
- 先查：是否必须登录、授权触发时机、code2session/jscode2session 服务端实现、anonymous_code 映射、openid/unionid 绑定、token 续期、拒权路径。
- 动作：用户动作后拉授权；取消授权可降级；服务端换 openid 并绑定业务用户；敏感响应脱敏。
- 证据：tt.login 返回脱敏样本、jscode2session/code2session 响应、anonymous_code 处理日志、会话过期复测、拒权截图。
- 兜底：只测已登录宿主不算通过。

### 7. tt.request / 域名 / 安全边界

- 适用：接口联调、域名白名单、鉴权失败、弱网、埋点、错误码。
- 先查：request 合法域名、HTTPS、证书、header、token、超时、重试、幂等、服务端日志、tt.setStorage/tt.getStorage、缓存 key 版本和用户维度；关闭合法域名校验只作为本地调试状态，不得作为体验版或线上证据。
- 动作：区分网络失败和业务失败；关键写操作带幂等；日志脱敏；弱网给可恢复 UI；本地只存非敏感、可过期、可重建数据，token、openid、手机号、支付参数、精确位置不得明文长期落盘；退出登录、切换账号、授权取消要清相关缓存。
- 证据：请求/响应脱敏样本、trace id、错误码矩阵、缓存命中/清理日志、弱网录屏。
- 兜底：Mock 通过不能替代体验版正式域名验证。

### 8. 支付 / 担保交易 / 交易组件

- 适用：tt.pay、pay-button、担保支付、商品订单、退款、结算、虚拟商品、交易组件。
- 先查：商品类型、iOS 限制、平台交易规则、服务端预下单、异步通知验签、重复回调、查单、退款状态机、结算状态和幂等键。
- 动作：前端只负责调起和展示；成功、取消、失败、超时、不确定态全部落订单状态；发货、退款、结算以后端确认为准；重复通知、重复退款和补偿任务必须幂等。
- 证据：订单号、支付参数脱敏样本、异步通知验签日志、查单/退款/结算结果、重复回调幂等记录。
- 兜底：客户端 success 不是发货证据。

### 9. 短视频 / 直播 / POI / 分享链路

- 适用：短视频挂载、navigateToVideoView、直播小雪花、openWebcastRoom、POI 入口、openEcGood、showDouyinOpenAuth、分享卡片、半屏小程序、返回视频。
- 先查：pay-button、openWebcastRoom、navigateToVideoView、openEcGood、showDouyinOpenAuth 的能力开通、tt.canIUse、场景值、query、落地页相关性、半屏高度、宿主支持、审核材料、定位授权、坐标系、逆地理、距离排序和地图层级。
- 动作：入口参数最小化并可追踪；挂载页与视频/直播/POI 强相关；位置能力在用户动作后触发并说明用途；无定位给手选城市/门店等降级；不支持宿主降级；返回路径明确。
- 证据：挂载入口录屏、场景值、query、半屏截图、定位拒权/允许截图、POI 或地图配置、后台配置或能力开通截图、专项 API 的宿主/基础库矩阵。
- 兜底：普通页面可用不代表挂载入口可用。

### 10. web-view / H5 回跳 / 外链

- 适用：web-view 承载、H5 登录态、外链、文件、回跳小程序。
- 先查：业务域名、HTTPS、白名单、state、签名、登录态桥接、返回路径、审核允许范围。
- 动作：只加载白名单 HTTPS；回跳校验来源、签名、state 和过期时间；失败页可恢复。
- 证据：域名配置、回跳参数、H5 与小程序会话日志、拒绝访问截图。
- 兜底：开发版域名可用不代表线上或审核可用。

### 11. 内容安全 / UGC / AI 生成内容

- 适用：评论、昵称头像、图片/视频上传、用户发布、AI 生成文案/图片、商品素材、客服消息。
- 先查：输入来源、上传格式/大小、是否公开展示、服务端审核、敏感词/涉政涉黄涉广、未成年人、版权、投诉和申诉入口。
- 动作：端侧只做格式、大小、时长、数量、临时文件生命周期和重复提交预校验；图片/视频选择、拍摄、压缩、上传、下载、预览要有进度、取消、失败重试和弱网恢复；内容安全以后端审核结果为准，审核中、驳回、申诉均有可恢复 UI。
- 证据：上传样本脱敏、审核结果、驳回原因、内容状态、投诉/申诉路径、审核备注。
- 兜底：本地预览通过不代表可公开展示或可过审。

### 12. 隐私合规 / 审核 / 发布

- 适用：权限声明、隐私协议、体验版、提审、驳回、灰度发布、全量上线。
- 先查：收集字段、权限用途、第三方 SDK、测试账号、审核备注、线上配置、监控和回滚。
- 动作：拒权可降级；审核备注写明入口和账号；体验版跑主链路；灰度观察启动成功率、登录成功率、授权拒绝率、支付成功/取消/未知态、挂载转化、接口错误率、JS 错误、首屏耗时、投诉/驳回。
- 证据：隐私协议链接、权限截图、体验版二维码、审核记录、灰度数据、监控阈值、回滚方案、客服可复现信息。
- 兜底：审核通过不等于线上稳定。

## 高频坑 / 防遗漏

### 高频坑

1. 把微信 wx.* 或支付宝 my.* 结论直接套到 tt.*。
2. 开发者工具可用就判完成，忽略基础库、真机和宿主版本。
3. app.json 漏 pages、tabBar、subPackages、preloadRule 或分享路径。
4. Page onShow 重入重复请求，返回页面覆盖用户刚改的数据。
5. Component detached 不清理监听、定时器、视频/直播上下文。
6. setData 传整页大对象或滚动中高频更新。
7. ttml/ttss 照搬 Web CSS，忽略安全区、半屏和原生组件层级。
8. 分包引用主包私有文件，开发版可用、线上加载失败。
9. tt.login 只处理 code，不处理 anonymousCode、openid 绑定和 session 过期。
10. tt.request 只测 Mock，未验证域名白名单、HTTPS、弱网和错误码。
11. tt.pay success 后立即发货，未等服务端回调/查单。
12. 视频、直播、POI、分享入口只测普通页面，不测挂载、半屏、场景值和审核相关性。
13. web-view 域名、H5 回跳来源、登录态桥接未验证。
14. 隐私合规、测试账号、权限用途、审核备注提审前才补。
15. 体验版通过后直接全量，缺灰度、监控、回滚和客服兜底。
16. 手机号、位置、相机、相册、麦克风、剪贴板等开放能力缺用户动作、用途说明、拒权降级或再次授权入口。
17. UGC/AI 内容只做本地预览，未接服务端内容安全、驳回/申诉状态和审核备注。
18. 本地缓存混放 token、openid、手机号、支付参数或精确位置，切换账号后污染数据。
19. 插件、第三方 SDK、交易/直播/地图组件未锁版本，升级后真机或审核异常。
20. Taro、uni-app、Remax 生成物未审，混入非 tt API、错误页面配置或平台插件版本。
21. pay-button、openWebcastRoom、navigateToVideoView、openEcGood、showDouyinOpenAuth 未列宿主/基础库矩阵。
22. 本地关闭合法域名校验后把调试结果当体验版或线上结果。
23. jscode2session 漏 anonymous_code，匿名态和登录态绑定断裂。
24. 异步通知、退款、结算缺验签和幂等，重复回调造成状态污染。
25. 只看总体包体，漏组件兼容和宿主差异。
26. 表单、键盘、字体缩放、textarea 层级只在调试器验收。

### 防遗漏清单

- 改路由：查 app.json、page json、tabBar、subPackages、preloadRule、分享路径、挂载路径、web-view 回跳；跨端查 project.tt.json、pages.json 与目标平台生成路径。
- 改组件：查 usingComponents、props、triggerEvent、样式隔离、生命周期、卸载清理。
- 改状态：查 setData 大小/频率、异步竞态、onShow 重入、缓存恢复、性能面板。
- 改登录：查 tt.login、anonymousCode、code2session、openid、token 续期、取消授权、隐私弹窗。
- 改接口：查 tt.request、域名白名单、鉴权、错误码、弱网、幂等、服务端日志、缓存版本、敏感信息落盘和清理；合法域名校验关闭只记为本地证据。
- 改交易：查 tt.pay、pay-button、担保交易、订单状态机、异步通知验签、重复通知、iOS 限制、退款/取消/结算幂等。
- 改挂载：查短视频、直播、POI、分享、场景值、半屏、落地页相关性、定位降级、审核材料，以及 openWebcastRoom、navigateToVideoView、openEcGood、showDouyinOpenAuth。
- 改内容：查 UGC/AI/商品素材、媒体文件格式大小、上传状态、服务端审核、投诉申诉、公开展示和审核备注。
- 改插件/SDK：查版本锁定、授权范围、隐私采集、包体、宿主兼容、拒权和低版本降级。
- 改发布：查体验版、审核、灰度发布、监控阈值、错误上报、回滚、客服话术。

## 约束

- 不重复全局触发规则；本技能只给抖音小程序原生端实现、排障和平台证据口径。
- 不把微信/支付宝小程序职责、通用 JS、后端 API 设计、UI 视觉方向、测试工程职责搬进本技能。
- 不凭调试器、Mock、单入口、单机型结论修改登录、支付、挂载、隐私、审核发布等高风险链路。
- 不新增无关重构、样式美化、架构升级或平台百科；内容安全、缓存、埋点、插件、定位、跨端生成物只保留端侧触发、证据、边界，复杂实现联动相邻技能。
- 不把客户端支付回调当最终交易状态。
- 不把体验版通过包装成审核通过，不把审核通过包装成灰度发布稳定。
- 涉测试结论遵守 tst 的已覆盖/部分覆盖/未覆盖/无法验证口径。
- 涉最终收口遵守 aud 的需求对账、影响面、证据、安全质量和剩余风险口径。

## 高频 Bug 反例库

- 反例 1：IDE/基础库假绿
  - 错法：开发者工具最新基础库正常，就说线上可用。
  - 对法：锁定目标基础库、真机调试和体验版复测，低版本用 tt.canIUse 降级。
  - 根因：调试器和线上最低基础库能力不一致。
- 反例 2：跨平台 API 误迁移
  - 错法：把 wx.login、wx.request 或 my.* 直接替换成 tt.* 后发布。
  - 对法：逐项查 tt API 参数、回调、宿主支持和授权规则。
  - 根因：不同小程序平台的 API 语义、审核和宿主能力不等价。
- 反例 3：页面入口漏枚举
  - 错法：只测首页跳转，漏测分享、短视频挂载、直播、POI 和 web-view 回跳。
  - 对法：按 app.json 与平台入口列矩阵，逐入口留 query 和录屏。
  - 根因：外部入口的场景值、容器和审核要求独立于普通路由。
- 反例 4：onShow 覆盖状态
  - 错法：把初始化放 Page onShow，返回页面重复拉接口并覆盖用户编辑。
  - 对法：onLoad 初始化，onShow 只做可重入刷新，异步请求带版本号。
  - 根因：冷启动、热启动、返回栈触发生命周期不同。
- 反例 5：Component 泄漏
  - 错法：attached 绑定全局事件，detached 不解绑视频、直播、计时器和监听。
  - 对法：Component 生命周期里成对注册/清理，上下文离页释放。
  - 根因：组件复用、分包切换和半屏返回会保留脏状态。
- 反例 6：setData 大包更新
  - 错法：滚动中 setData 整个列表或整页状态。
  - 对法：路径式更新变更项，分页/虚拟列表，合并异步状态。
  - 根因：逻辑层到视图层通信成本高，数据越大越卡。
- 反例 7：ttml/ttss 照搬 H5
  - 错法：用复杂选择器、固定高度、z-index 盖视频或地图原生组件。
  - 对法：按 ttml/ttss 与原生组件层级设计，安全区、暗黑和半屏单独适配。
  - 根因：小程序渲染层不是完整浏览器 CSS 环境。
- 反例 8：分包路径漏配
  - 错法：新增分包页面但 app.json subPackages、资源或 preloadRule 漏配。
  - 对法：同步注册路径、资源和预加载，体验版弱网首进验证。
  - 根因：开发跳转和线上包加载边界不同。
- 反例 9：登录只换 code
  - 错法：只把 code 发服务端，忽略 anonymousCode、openid 绑定和取消授权。
  - 对法：区分匿名态/登录态，经 code2session 换 openid，拒权可降级。
  - 根因：宿主登录态与业务登录态不是一回事。
- 反例 10：tt.request 域名未验
  - 错法：Mock 接口通了就发布，线上域名白名单或 HTTPS 不通过。
  - 对法：真机/体验版访问正式域名，记录 request id、错误码和弱网兜底。
  - 根因：网络权限、证书和域名配置只在真实环境暴露。
- 反例 11：tt.pay success 立即发货
  - 错法：客户端 success 后直接改订单成功。
  - 对法：服务端回调验签/查单确认后发货，重复回调幂等。
  - 根因：客户端只证明收银台返回，不证明资金最终确认。
- 反例 12：担保交易边界混乱
  - 错法：前端自己拼交易参数、退款和履约状态，不查平台交易规则。
  - 对法：由服务端维护订单、退款、履约和回调状态机，前端只展示状态。
  - 根因：交易合规、验签、幂等和 iOS 限制不属于纯前端事实。
- 反例 13：视频/直播/POI 只测普通入口
  - 错法：普通页面能播放视频，就认为短视频挂载、直播半屏和 POI 可用。
  - 对法：用真实挂载入口、场景值、半屏高度、返回路径和相关性逐项验证。
  - 根因：挂载容器、入口参数和审核相关性独立于普通页面。
- 反例 14：分享参数污染
  - 错法：分享 query 带敏感 token 或无限期业务参数。
  - 对法：分享参数最小化，使用短码、签名、过期时间和兼容旧参数。
  - 根因：外部入口参数会传播、篡改、过期或丢失。
- 反例 15：web-view 信任回跳
  - 错法：web-view H5 任意 URL 回跳小程序并带业务参数。
  - 对法：限制白名单 HTTPS，校验来源、签名、state、过期和登录态。
  - 根因：web-view 是跨域边界，不能信任 H5 输入。
- 反例 16：开放能力授权前置弹窗
  - 错法：页面加载立即索要手机号、位置、相机或相册权限。
  - 对法：在用户动作后解释用途，拒权有替代流程或明确中断。
  - 根因：抖音审核和隐私合规要求最小必要、用途明确、可拒绝。
- 反例 17：真机调试缺失
  - 错法：模拟器和预览通过，没有 iOS/Android 真机日志。
  - 对法：至少覆盖目标宿主、系统和基础库，留截图/录屏/日志。
  - 根因：设备权限、键盘、安全区、系统 WebView 差异只在真机暴露。
- 反例 18：审核材料不含入口
  - 错法：审核备注只写修复问题，测试账号、挂载路径和支付方式不清。
  - 对法：写明账号、入口、核心路径、支付/权限测试方式和限制说明。
  - 根因：审核员无法复现会按不可用、诱导或能力不相关驳回。
- 反例 19：灰度发布无监控
  - 错法：审核过后直接全量，支付失败率或挂载投诉升高才发现。
  - 对法：灰度分批观察登录、支付、挂载、崩溃、投诉和回滚指标。
  - 根因：审核只看准入，不能证明线上稳定。
- 反例 20：AI/低代码生成未审计
  - 错法：生成代码里混入 wx.*、DOM、Web CSS、未授权采集和大对象 setData。
  - 对法：按 tt API、ttml/ttss、权限、setData、入口和审核清单逐条审计。
  - 根因：通用生成器默认模式常不是抖音小程序原生语义。
- 反例 21：跨端生成物假一致
  - 错法：Taro、uni-app、Remax 编译通过就认为抖音端可用。
  - 对法：审 Taro @tarojs/plugin-platform-tt/project.tt.json、uni mp-toutiao/pages.json、Remax @remax/toutiao、目标 app.json 与 tt API 差异。
  - 根因：跨端适配层会改路由、组件、API 和包体，编译成功不等于端能力匹配。
- 反例 22：专项 API 无矩阵
  - 错法：pay-button、openWebcastRoom、navigateToVideoView、openEcGood 或 showDouyinOpenAuth 在单宿主可用就上线。
  - 对法：列 tt.canIUse、宿主版本、基础库、iOS/Android、开通状态和降级入口。
  - 根因：开放能力受宿主、基础库和后台权限共同约束。
- 反例 23：交易通知不幂等
  - 错法：收到一次异步通知就直接改发货、退款或结算状态。
  - 对法：服务端验签、查单、状态机和幂等键闭环，重复通知只记录不重复履约。
  - 根因：交易通知可能重复、乱序、延迟或补偿触发。
- 反例 24：app secret 放前端
  - 错法：为了省后端联调，把 app secret、支付密钥或签名 token 写进 js、project 配置、日志或审核截图。
  - 对法：密钥只在服务端，前端只拿短期业务 token 和脱敏响应；日志、截图、审核材料都做敏感信息清理。
  - 根因：小程序包、日志和截图都可能被反编译、上传或外部可见。
- 反例 25：客户端 openid 当身份
  - 错法：前端缓存 openid 后直接传给接口，服务端按 openid 判断用户和权限。
  - 对法：服务端用 code2session/jscode2session 建立业务会话，接口只信服务端 session 和权限校验。
  - 根因：客户端字段可伪造、过期、串号或被旧缓存污染。
- 反例 26：金额由前端裁定
  - 错法：前端根据页面价格拼订单金额，支付 success 后按前端金额发货。
  - 对法：服务端按商品、优惠、库存和订单状态生成金额，支付后验签、查单、核金额再履约。
  - 根因：价格、优惠和交易状态属于服务端事实，前端只负责展示和调起。
- 反例 27：只测 IDE happy path
  - 错法：开发者工具登录、授权、支付 Mock 都通了，就报完成。
  - 对法：补 iOS/Android 真机、体验版、弱网、拒权、取消支付、重复点击、旧缓存和挂载入口证据。
  - 根因：调试器、Mock、正式域名、宿主权限和体验版容器不是同一环境。
- 反例 28：授权拒绝无路可走
  - 错法：用户拒绝手机号、位置、相机或相册后页面白屏或无限弹窗。
  - 对法：每个权限都有用途说明、拒权降级、重新授权入口和业务中断提示。
  - 根因：平台审核要求可拒绝，真实用户也会拒权或系统禁用权限。
- 反例 29：API 字段直灌 UI
  - 错法：tt.request 返回什么就直接 setData，字段缺失或类型变化导致白屏。
  - 对法：做 response schema、默认值、旧字段兼容、错误态和 trace id；高风险接口联动 API 契约。
  - 根因：小程序端会遇到灰度接口、旧缓存、弱网重放和后端兼容窗口。
- 反例 30：审核材料补不齐
  - 错法：提审只写功能描述，没有测试账号、支付路径、挂载入口、权限用途和限制说明。
  - 对法：审核备注列账号、入口、步骤、测试数据、支付/退款方式、权限触发点和特殊宿主要求。
  - 根因：审核员不能复现时会按不可用、诱导授权或能力不相关驳回。
- 反例 31：缓存跨账号污染
  - 错法：本地缓存用户资料、门店、订单草稿和 token，切换账号后继续复用。
  - 对法：缓存 key 带用户维度和版本；退出登录、授权取消、token 失效、切换账号时清理。
  - 根因：抖音宿主账号和业务账号会变化，旧缓存会造成越权、错单和隐私泄露。
- 反例 32：挂载入口参数不验
  - 错法：短视频/直播/分享 query 直接带业务 id 进入详情页，不校验来源、签名和过期。
  - 对法：使用服务端短码或签名参数，校验 scene、来源、过期和业务可见性，不合法进入降级页。
  - 根因：外部入口参数会传播、篡改、丢失、过期，不能当可信输入。
- 反例 33：内容发布只看上传成功
  - 错法：图片/视频上传成功就公开展示，未处理审核中、驳回、申诉和投诉。
  - 对法：端侧展示上传进度和状态，公开展示以后端内容安全和业务审核结果为准。
  - 根因：上传成功只证明文件到达，不证明内容合规或可公开。
- 反例 34：体验版当灰度稳定
  - 错法：体验版主链路跑通后直接全量上线，没有监控和回滚。
  - 对法：灰度观察启动、登录、授权、支付、挂载、接口错误、JS 错误、投诉和回滚路径。
  - 根因：体验版样本少，无法覆盖宿主版本、网络、基础库和真实用户行为。

## 2024-2026 新坑速查

- 2024-2026 基础库漂移：开发者工具默认新基础库更宽松，线上最低基础库缺 API；修法是锁目标版本、tt.canIUse、真机和体验版复测。
- 2024-2026 半屏常态化：短视频、直播、POI 挂载更常以半屏承载；修法是首屏高度、返回视频、支付反馈和相关性单独设计。
- 2024-2026 交易链路收紧：支付、退款、担保交易、交易组件和虚拟商品更重视服务端闭环；修法是订单状态机、回调验签、幂等和 iOS 限制前置。
- 2024-2026 隐私合规前置：手机号、位置、相册、摄像头、麦克风、第三方 SDK 采集要提前说明；修法是最小采集、拒权降级、隐私协议可达。
- 2024-2026 web-view 边界收紧：H5 登录态、跳转、文件下载和外链更易被拦；修法是白名单、state、签名、来源校验和错误页。
- 2024-2026 分享与挂载参数污染：外部入口 query 可能过期、篡改、丢失；修法是服务端短码、签名、过期时间和兼容旧参数。
- 2024-2026 设备兼容差异：摄像头、麦克风、相册、键盘、安全区、系统 WebView 与调试器不同；修法是真机矩阵和拒权录屏。
- 2024-2026 多宿主矩阵：抖音、抖音极速版、头条/西瓜等目标宿主需列最低版本、基础库、关键 tt API、半屏/挂载/支付/定位/相机/直播支持；不支持时隐藏入口、降级页或升级提示，不能白屏。
- 2024-2026 审核规范复现成本上升：审核员需要明确账号、入口、路径、能力开通和限制说明；修法是审核备注写清步骤和测试数据。
- 2024-2026 灰度不可省：审核通过后仍可能因宿主、网络、基础库分布出错；修法是分批灰度、监控支付/登录/挂载和快速回滚。
- 2024-2026 AI/低代码生成风险：常混入 wx.*、Web CSS、异步状态、权限弹窗、UGC 审核和本地缓存错误；修法是按本技能清单逐条审计。
- 2024-2026 跨端生成物风险：Taro @tarojs/plugin-platform-tt/project.tt.json、uni mp-toutiao/pages.json、Remax @remax/toutiao 可能改路由、组件、包体和 API 语义；修法是审生成物、目标配置和真机差异。
- 2024-2026 专项能力矩阵：pay-button、openWebcastRoom、navigateToVideoView、openEcGood、showDouyinOpenAuth 要列 tt.canIUse、宿主、基础库、系统、后台权限和降级。
- 2024-2026 交易后端闭环：jscode2session anonymous_code、异步通知验签、退款、结算、幂等必须与服务端日志对齐；端侧只展示状态。
- 2024-2026 包体与组件兼容：主包、分包、插件、原生组件和跨端组件要给体积与兼容矩阵，低版本宿主不支持时降级。
- 维护状态风险：跨端平台插件和抖音端专项 API 维护节奏不同，长期未更新或低活跃依赖要标版本锁定、替代方案和验证缺口。

## 与相邻技能的边界

- 本技能负责：抖音小程序原生 ttml/ttss/js/json、Page、Component、setData、app.json、project.config.json、project.tt.json、pages.json、tt API、端能力、页面入口、开放能力授权、短视频、直播、POI、分享、web-view、支付/担保交易、跨端生成物端侧审计、内容发布端侧状态、缓存边界、插件/SDK 端侧核查、审核规范、真机与宿主差异的定位和证据口径。
- JavaScript/TypeScript 开发/javascript-typescript-development（jsts） 负责：复杂 JS/TS 语言实现、类型、依赖、构建、工程化和通用前端逻辑质量；本技能只要求其实现符合抖音端能力和证据边界。
- API 工程/api-engineering（api） 负责：tt.request 背后的接口契约、认证语义、错误码、兼容策略、幂等和后端字段；本技能只核对端侧调用、白名单和响应证据。
- UI 设计实现/ui-design（u） 负责：视觉风格、布局审美、文案和交互体验；本技能只核对 ttml/ttss、原生组件层级、安全区、半屏和真机可用性。
- 品牌与视觉方向/brand-visual-direction（b） 负责：设计方向、品牌一致性和体验策略；本技能不产出视觉方向，只把平台容器限制反馈给设计侧。
- 产品经理/product-manager（pm） 负责：需求范围、用户路径、业务优先级和验收口径；本技能只补抖音入口、审核、权限和交易约束。
- Web 安全/web-security（wsec） 负责：跨域、XSS、CSRF、token、签名、内容安全、web-view/H5 安全模型；本技能只指出小程序端输入、回跳、白名单、敏感日志和本地存储风险。
- 测试验证/test-engineering（tst） 负责：测试矩阵、回归、真机/体验版/线上验证和无法验证口径；本技能提供抖音端必须覆盖的入口与证据点。
- 代码审计/code-audit（aud） 负责：改动完成前的需求对账、影响面、安全质量、证据缺口和剩余风险；本技能提供抖音小程序专属审计清单。
