# 支付宝小程序

支付宝小程序（alipay-miniprogram，兼容 slug: almp）负责本技能描述范围内的定位、执行、验证和交接边界；旧短 slug 仅作兼容 alias/URL 主键，不作为规范技能名。

定位：本技能只处理支付宝小程序原生链路的实战排障与交付收口，定位改为原生优先+跨端产物核验。目标不是把微信、抖音、H5、uni-app、Taro、Mpx、Remax 经验替换前缀，而是在 AXML、ACSS、Page、Component、SJS、my API、IDE 真机预览、授权登录、商家开放能力、分包、审核灰度、端内 WebView、支付联动和跨端构建产物中先拿到证据，再做最小修改。

## 快速总则

1. 先定端和版本：记录小程序开发者工具版本、模拟器基础库、支付宝 App 真机基础库、Android/iOS、体验版、审核版、灰度版；IDE 通过不等于真机通过。
2. 先定入口：app.json、mini.project.json、页面级 json、pages、tabBar、subPackages、usingComponents、插件、web-view 域名、订阅消息落地页、二维码/分享/生活号/商家入口都要核。
3. 先定平台语法：模板用 AXML 的 a:if、a:for、a:key；样式用 ACSS；事件用 onTap/catchTap；组件通信用 props 回调；API 用 my.*。
4. 先定原生基线：优先确认原生 app/page/component json、axml/acss/js 产物是否正确，再判断跨端框架配置、编译插件、条件编译和运行时适配。
5. 先定跨端来源：Taro、uni-app、Mpx、Remax、AntD Mini 等项目必须记录源码框架、构建命令、目标平台、插件版本、产物目录和 CI 环境；不能只看源码层配置。
6. 先定授权边界：my.getAuthCode 的 scopes 是字符串；authCode 交服务端换态；前端不得保存支付宝 access_token、手机号解密结果、支付密钥。
7. 先定标识层级：user_id、open_id、业务 uid、会员 id、商家门店 id 分层；跨主体、生活号、会员、商家小程序不得混用。
8. 先定支付边界：小程序只调起 my.tradePay 或承接 JSAPI 支付结果；tradeNO 大小写固定；9000 也要服务端查单，8000/6004/取消/网络错误必须轮询或查单。
9. 先定合规能力：隐私协议、授权弹窗、订阅消息模板、开放能力、商家能力、域名白名单、插件权限、测试账号与开放平台配置一致。
10. 先定能力兼容：新 API、插件、WebView、相册、摄像头、定位、蓝牙、扫码、文件、地图等能力先探测再调用；记录最低基础库和支付宝 App 版本；不支持时给降级 UI、灰度开关或服务端兜底。
11. 先定证据：每个结论绑定页面路径、基础库、端、操作步骤、日志/截图、请求响应、授权记录、订单记录、入口样本、审核记录或灰度指标。
12. 先定联动：改接口契约找 api；改支付资金态找 ali；改安全找 wsec；改发布找 rls；涉测试找 tst；收口找 aud。
13. 先定最小改动：修 Bug 必须复现、最小修改、原路径复验；证据不足时先不改。

## 单技能工程门禁

### 开发前门禁
- 明确本次改动属于页面、组件、登录、手机号、my.request、支付调起、订阅消息、开放能力、WebView、分包、审核、灰度中的哪一类；不能只说“加个支付宝小程序功能”就直接写代码。
- 明确目标端：IDE、模拟器、Android 真机、iOS 真机、体验版、审核版、线上版；只测 IDE 的结论不能作为完成证据。
- 明确入口：普通页面跳转、tabBar、扫码、分享、订阅消息、生活号、商家入口、端内 WebView 回跳；所有入口参数默认不可信。
- 明确账号和主体：appId、商家主体、会员主体、门店、生活号、支付主体、测试账号和环境；主体不清时不改授权、手机号、会员、支付和开放能力。
- 明确服务端边界：authCode 换态、手机号解密、订单金额、tradeNO、notify 验签、查单、模板消息发送、开放能力服务端配置都不放前端。
- 明确 API 契约：字段新增、字段缺失、类型变化、旧客户端兼容、错误码、trace id、灰度接口和降级文案；禁止把返回对象直接灌 UI。
- 明确验收证据：页面路径、基础库、端、操作步骤、真机截图或录屏、请求响应、服务端日志、订单记录、审核材料、灰度指标。

### 开发中门禁
- AXML、ACSS、SJS、Page、Component、my API、页面 json 和 app.json 必须一起核；不能只改 JS 逻辑。
- setData 只更新渲染需要的最小字段；长列表、表单输入、滚动、定时器和 WebSocket 回调不能整对象刷 UI。
- 登录和手机号必须有授权拒绝、重新授权、过期、弱网、切号、退出登录路径；拒绝授权不能卡死非必要流程。
- my.request 必须保留 HTTP 状态、业务码、错误码、trace id、原始错误和超时；API 字段缺失或类型漂移要有兼容兜底。
- 订阅消息必须绑定用户动作、模板用途、落地页和授权结果；模板审核状态和失败回调要能追踪。
- WebView URL、回跳参数、短期票据、来源页面和登录态必须可验证；不能信任 H5 自带 Cookie 或任意 returnUrl。
- 分包、插件、组件库和跨端产物必须核最终支付宝 app/page/component json、axml/acss/js；禁止只看源码 import 或 IDE 编译成功。

### 发布前门禁
- 至少给出 Android/iOS 真机或体验版证据；没有设备、账号、商家权限、支付环境或能力开通时，明确标“无法验证”。
- 支付只验证调起不算完成；必须有服务端订单态、notify 或查单记录、重复点击、取消、处理中、弱网回流证据。
- 审核材料必须能复测：测试账号、商家账号、支付环境、门店数据、权限用途、隐私协议、审核路径和截图与当前版本一致。
- 灰度发布必须有指标阈值和回滚口径：登录成功率、授权拒绝率、接口错误率、支付转化、订阅到达率、WebView 错误率。
- 完成前按本技能输出要求列影响链路和剩余风险；证据缺口不能包装成“已完成”。

## 硬禁止与低级错拦截

- 禁止把 app secret、支付宝 access_token、手机号解密、支付密钥、支付金额裁定、notify 验签、查单逻辑放在小程序前端。
- 禁止把前端传入的金额、优惠、会员等级、门店、订单状态当可信事实；金额和资金态只能由服务端、支付通知和查单裁定。
- 禁止只信 my.tradePay 客户端 resultCode；9000、8000、6004、取消、网络错误、回流丢失都要服务端确认。
- 禁止只测 IDE、只测模拟器、只测 H5、只测 happy path；支付宝小程序交付必须有真机或体验版证据。
- 禁止不处理授权拒绝、手机号拒绝、定位/相册/摄像头拒绝、订阅消息拒绝；每个拒绝路径都要继续可用或给明确降级。
- 禁止忽略审核材料、隐私协议、权限用途、测试账号、商家能力开通和域名白名单；审核失败不是发布后再补。
- 禁止把 wx/tt/浏览器 API 习惯直接搬过来；发现 wx:、bindtap、triggerEvent、localStorage、document/window 要先停下排查平台差异。
- 禁止在日志、埋点、缓存、source map、错误上报中保存手机号、身份证、定位、相册 URL、长期票据、订单敏感字段和支付参数。
- 禁止 API 字段不兼容时直接崩溃；旧字段、新字段、空值、字符串数字、布尔字符串、数组缺失都要有兼容和错误态。
- 禁止为了赶进度直接手改 dist/generated 产物；必须修生成配置、平台条件编译、插件版本或构建命令，并记录回滚。

## 场景执行卡

### 1. IDE 真机预览 / 基础库 / 项目配置 / 构建产物
- 适用：预览二维码失败、IDE 正常真机白屏、基础库兼容、插件加载失败、体验版行为不同、CI 产物异常、跨端源码正常但支付宝产物异常。
- 先查：IDE 版本、支付宝 App 版本、基础库、app.json、mini.project.json、页面级 json、编译模式、分包、插件版本、域名白名单、构建命令、环境变量、npm/插件版本、压缩配置、source map、资源 CDN、包体积和主包预算。
- 动作：建最小复现页；清缓存重编译；对比 IDE、模拟器、真机调试、体验版；确认 API 是否高于最低基础库；确认 IDE 本地、CI 构建、体验版产物同源可追溯；禁止把开发域名、测试 appId、测试支付配置、未脱敏 sourcemap 带入审核版或线上版。
- 产物核验：逐项打开 app/page/component json、axml/acss/js，核 pages、tabBar、subPackages、usingComponents、mycomponents、component2、styleIsolation、插件声明、权限声明和资源路径是否由构建链正确生成。
- 证据：控制台、真机日志、预览二维码、基础库截图、失败页面路径、复现视频、构建日志、包体积截图、产物版本号。

### 2. 跨端框架到支付宝产物核验
- 适用：Taro、uni-app、Mpx、Remax、AntD Mini 或多端组件库接入支付宝后，出现构建失败、页面白屏、组件缺失、样式错乱、API 不支持、插件不生效。
- 先查：源码框架和版本、构建命令、目标平台、插件清单、条件编译、生成目录、依赖锁文件、CI Node/npm/pnpm 版本、是否二次加工产物。
- Taro：核 plugin-platform-alipay 是否启用，构建命令是否为 taro build --type alipay，产物中 AXML/ACSS、页面 json、usingComponents 和插件声明是否符合支付宝规则。
- uni-app：核 mp-alipay 目标、MP-ALIPAY 条件编译、manifest/pages 配置、组件和 API 适配层，确认生成的 app/page json、axml/acss/js 与源码预期一致。
- Mpx：核 mode ali、srcMode、平台差异配置、组件转换、样式隔离和运行时注入，确认 dist 产物不是微信或其他平台残留。
- Remax：核 target ali、remax/ali 运行时、页面配置、组件映射和构建插件，确认事件、生命周期、样式和 API 产物符合支付宝。
- AntD Mini：核 usingComponents/mycomponents、组件 json、样式引入、component2、基础库要求和按需引入；不要只看源码 import 成功。
- 动作：先比对原生最小页，再比对框架产物；产物问题优先回溯插件/配置/条件编译，不直接手改 generated 目录；若必须临时补丁，记录生成命令和回滚方式。
- 证据：框架版本、构建命令、产物目录、关键 json/axml/acss/js 截图或摘录、真机日志、包体积、差异说明。

### 3. AXML / ACSS / SJS / 迁移交互
- 适用：微信迁移、事件不触发、样式不生效、slot 异常、SJS 报错。
- 先查：wx:if、wx:for、bindtap、catchtap、triggerEvent、.wxss、wxs、dataset 使用是否残留。
- 动作：改 a:if/a:for/a:key、onTap/catchTap、ACSS、import-sjs；组件通信改 this.props.onXxx；样式隔离按页面 json 复核。
- 证据：渲染截图、事件日志、组件 json、样式对比、真机录屏。

### 4. Page / Component 生命周期
- 适用：onLaunch/onShow/onLoad/onReady/onHide、参数丢失、返回不刷新、组件 props 不更新。
- 先查：入口 query 类型、页面栈、返回来源、组件复用、异步 props、全局状态初始化。
- 动作：onLaunch 轻量；onLoad 只解析入口参数；onShow 处理回流；Component 用 props 回调；异常分支 setData 兜底。
- 证据：生命周期日志、query 样本、props 变化、页面栈、重复进入结果。

### 5. setData / 首屏 / 列表性能
- 适用：状态不同步、滚动卡顿、首屏慢、长对象更新、图片富文本膨胀。
- 先查：setData 频率、payload 大小、深层对象、长列表、首屏同步任务、图片资源体积。
- 动作：只传变化字段；分页或虚拟化；合并高频更新；非渲染数据放实例字段；onLaunch 不阻塞首屏。
- 证据：payload、耗时、首屏时间、滚动录屏、弱网表现、低端 Android 表现。

### 6. 分包 / 路由 / tabBar / 首屏体积 / 入口参数
- 适用：新增页面、分包迁移、包体积超限、跳转失败、tabBar 异常、二维码入口打不开、分享或外部拉起异常。
- 先查：pages、subPackages、tabBar、preloadRule、分包根目录、主包资源、路径大小写、参数编码、scene/query、分享参数、生活号/商家入口、scheme/URL 拉起。
- 动作：tabBar 页留主包；非首屏大模块进分包；高频页预加载；跳转路径和二维码参数逐项复核；分享、二维码、小程序码、生活号、商家入口、scheme/URL 都按不可信入口处理，参数只当字符串，先解码再校验白名单；订单、门店、活动、渠道参数优先签名或服务端换短码；过期、篡改、重复进入给兜底页。
- 证据：包体积、路由表、分包首次进入耗时、冷启动数据、入口样本、scene/query 解码结果、签名校验日志、渠道归因、异常入口降级截图。

### 7. 授权登录 / 会员 / 用户标识
- 适用：静默登录、用户资料、手机号、会员绑定、商家会员、user_id/open_id 混乱。
- 先查：auth_base/auth_user、my.getAuthCode、服务端换态、会员映射、隐私说明、拒绝授权路径。
- 动作：scopes 用字符串；authCode 立即给服务端；user_id/open_id/业务 uid/会员 id 分层；拒绝授权不阻断非必要链路。
- 证据：授权弹窗、换态日志、用户映射、拒绝/重新授权路径、服务端 session 记录。

### 8. 本地存储 / 会话 / 多账号
- 适用：登录串号、退出后仍有旧态、体验版和线上缓存污染、弱网重复登录、用户切换异常。
- 先查：my.getStorage/my.setStorage key、TTL、环境前缀、账号主体、业务 uid、会员 id、token/session 过期策略。
- 动作：敏感信息不落本地；缓存 key 带环境和用户维度；退出登录清理身份态；authCode 换态失败不复用旧 session；弱网重试要幂等。
- 证据：缓存 key 列表、登录/退出/切号录屏、session 过期日志、弱网重试结果。

### 9. my.request / 域名 / API 契约 / unsupported API
- 适用：真机接口失败、证书/域名、header 丢失、灰度接口、错误码不一致、跨端 API 在支付宝不支持、适配层吞错。
- 先查：HTTPS 证书、域名白名单、环境配置、header、网关鉴权、超时、错误码、trace id、API 兼容表、基础库版本、框架 API proxy 实现。
- 动作：封装保留 status、header、result、trace id；错误分层；unsupported API 先探测、降级或服务端兜底；API proxy 不得隐藏支付宝原始错误码、回调参数和时序；契约变化联动 api；安全头和跨域由 wsec 复核。
- 证据：请求响应样本、状态码、trace id、服务端日志、真机网络面板、兼容探测结果、proxy 前后日志。

### 10. WebSocket 单连接 / 回调时序
- 适用：实时消息丢失、重复连接、页面切换后收不到消息、onOpen/onMessage/onError/onClose 顺序异常、跨端 socket 封装和支付宝真机表现不一致。
- 先查：是否全局只维护一个 WebSocket 单连接、连接状态机、重连退避、页面订阅关系、前后台切换、弱网/断网、token 刷新和服务端心跳。
- 动作：明确连接所有权；避免每页重复 connect；按支付宝回调时序记录 onOpen、onMessage、onError、onClose；重连要幂等且解绑旧监听；进入后台降级或暂停非必要订阅；消息 ack 和补偿由服务端配合。
- 证据：连接 id、页面路径、回调时间线、弱网录屏、服务端连接日志、重复连接计数、消息补偿记录。

### 11. my.tradePay / JSAPI 支付联动边界
- 适用：收银台调起失败、状态错判、重复支付、结果页、H5/端内 WebView 支付回跳。
- 先查：服务端下单、tradeNO、notify 验签、查单、幂等键、订单状态机、JSAPI 调起端。
- 动作：小程序只下单和调起；所有 resultCode 回服务端确认；防重复点击；处理中可回查；资金态由 ali 和后端裁定。
- 证据：tradeNO、resultCode、订单号、通知日志、查单记录、重复点击测试、回跳 URL。

### 12. 模板消息 / 订阅消息 / 开放能力
- 适用：消息不触达、模板审核失败、订阅授权失败、生活号/商家开放能力异常。
- 先查：模板 id、业务用途、触发动作、落地页、用户授权、开放平台能力开通、商家主体和门店权限。
- 动作：只在用户动作后申请订阅；模板文案与业务一致；落地页可访问；开放能力缺权限时给降级。
- 证据：授权截图、消息记录、模板配置、开放平台能力截图、失败回调。

### 13. 端内 WebView / 插件 / H5 联动
- 适用：web-view 白屏、登录态断裂、回跳被劫持、插件能力异常、H5 与小程序状态不一致。
- 先查：web-view 域名白名单、URL 参数、短期票据、回跳协议、插件 appId/version/权限、第三方 Cookie 限制。
- 动作：URL 白名单和签名校验；短期票据换登录态；回跳验签；插件锁版本并准备降级；安全问题联动 wsec。
- 证据：URL、票据生命周期、回跳日志、插件版本、真机失败回调。

### 14. 商家小程序 / 审核 / 灰度发布
- 适用：提审驳回、商家能力不可用、测试账号不可用、体验版和线上不一致、灰度异常。
- 先查：商家主体、类目、能力包、隐私协议、测试账号、支付环境、门店/会员配置、审核材料。
- 动作：提审前跑登录、核心功能、支付、订阅消息、web-view、插件、拒绝授权；体验版真机复测；灰度看错误率、支付转化、授权拒绝率。
- 证据：审核材料、测试账号、体验版版本号、真机录屏、灰度指标、回滚方案。

### 15. 原生组件 / 表单 / 弹层 / 滚动
- 适用：input 失焦、键盘遮挡、picker 异常、form 提交缺字段、地图/扫码/上传失败、弹层层级错乱、scroll-view 卡顿。
- 先查：组件是否为支付宝原生组件、页面 json 配置、表单 name、受控/非受控状态、键盘高度、fixed/z-index、滚动容器高度、权限授权。
- 动作：建立最小交互页；输入类组件只同步必要字段；弹层避免遮挡原生组件；上传/扫码/定位提供拒绝授权降级；低端 Android 真机复测。
- 证据：交互录屏、组件配置、表单 payload、授权状态、失败回调、端差异截图。

### 16. 文件 / 媒体 / 位置 / 设备权限
- 适用：相册、摄像头、上传下载、定位、蓝牙、剪贴板、设备信息、通讯地址、文件预览或临时文件异常。
- 先查：调用时机、用途说明、授权状态、隐私协议、文件类型/大小/数量、临时文件生命周期、上传归属、失败回调。
- 动作：最小化采集；调用前说明用途；拒绝后继续非必要流程；上传前校验类型、大小、数量和业务归属；临时文件只短期使用；日志与埋点不得记录原始手机号、身份证、定位明文、相册文件 URL 或长期票据。
- 证据：授权说明截图、拒绝授权路径、上传 payload、文件校验日志、临时文件清理记录、脱敏埋点样本。

### 17. 观测 / 埋点 / 灰度证据
- 适用：线上偶发、审核复现困难、灰度转化异常、支付/授权/WebView 问题无法定位。
- 先查：trace id、session id、页面路径、入口来源、基础库、端、网络、用户授权态、接口错误码、支付订单号、WebView URL。
- 动作：前端错误、性能、业务事件统一带 trace id；敏感字段脱敏；灰度设阈值和回滚开关；关键链路埋点与服务端日志可关联；缺观测先补证据口径，不凭猜测下结论。
- 证据：埋点样本、服务端 trace、灰度看板、告警记录、回滚演练结果。

## 高频坑 / 防遗漏

### 高频坑
1. 只改 my API，AXML 指令、ACSS、事件名仍是微信写法。
2. 小程序开发者工具预览通过，却未跑 Android/iOS 真机和体验版。
3. onLoad query 把字符串当 number/boolean，二维码入口和订阅消息入口异常。
4. Component 继续用 triggerEvent，父组件收不到事件。
5. setData 整页大对象导致首屏慢和滚动卡顿。
6. tabBar 页面放进分包或路径大小写不一致。
7. my.getAuthCode scopes 传数组或 auth_base/auth_user 混用。
8. user_id、open_id、业务 uid、会员 id、商家门店 id 混存。
9. my.request 在 IDE 正常，真机因域名、证书、header、灰度环境失败。
10. my.tradePay 只看客户端 resultCode，不查单不验签。
11. 模板消息/订阅消息未绑定用户动作或落地页不可访问。
12. web-view 域名、登录态、回跳签名未校验。
13. 插件版本未锁、开放能力未开通、商家权限未配置。
14. 隐私协议与实际采集字段不一致，2024-2026 审核直接驳回。
15. 体验版、审核版、线上版环境、测试账号和支付配置不一致。
16. 页面级 json、usingComponents、插件声明或权限声明漏配，只看全局 app.json。
17. 新 API、插件、WebView、定位/蓝牙/扫码/文件能力未做版本门槛和降级。
18. 本地缓存不带环境/用户维度，退出登录或切号后串 session。
19. 分享、二维码、scheme/URL 拉起参数未校验，被污染、回放或归因错乱。
20. 灰度缺 trace/session/page-view/event/error/performance 关联，线上偶发无法定位。
21. Taro 未启用 plugin-platform-alipay 或未用 taro build --type alipay，却拿其他平台产物排查支付宝问题。
22. uni-app 忘记 mp-alipay 或 MP-ALIPAY 条件编译，导致 API、组件和样式分支错进产物。
23. Mpx mode ali/srcMode 配置错，产物混入非支付宝平台语法。
24. Remax target ali/remax/ali 运行时不匹配，生命周期、事件或组件映射失真。
25. AntD Mini 只看源码 import，未核 usingComponents/mycomponents、component2 和基础库要求。
26. unsupported API 被 API proxy 吞错，真机只表现为无响应或回调时序错乱。
27. WebSocket 每页各连一次，后台切换或弱网后重复连接、消息乱序。

## 约束

- 不把支付宝小程序当微信小程序换前缀。
- 不用 IDE 预览替代真机调试、体验版、审核验证。
- 不在前端保存支付宝 access_token、支付密钥、手机号解密结果、敏感证件或长期票据。
- 不把 my.tradePay 客户端返回当资金最终态。
- 不越权替代服务端支付、数据库、发布策略、安全审计；按需联动相邻技能。
- 不跳过隐私合规、授权弹窗、模板消息用途、开放能力资质和审核材料。
- 不把入口参数、分享参数、二维码参数、WebView URL、scheme/URL 拉起当可信输入。
- 不在日志、埋点、缓存、source map 中泄露手机号、证件、定位明文、长期票据、支付或鉴权敏感信息。
- 不直接手改 generated/dist 产物替代修框架配置、插件版本、条件编译或生成命令；确需临时补丁必须有回滚记录。
- 不让 API proxy 吞掉 unsupported API、原始错误码、回调参数或时序证据。
- 不做无关重构；修 Bug 先复现、最小修改、原路径复验。
- 不输出无证据结论；缺基础库、端、入口、日志时标证据不足。
- 正文不得包含 fenced code block。

## 高频 Bug 反例库

- 反例 1：IDE 假绿。错法：小程序开发者工具预览正常就判线上可用。对法：记录 IDE、基础库、Android/iOS 真机、体验版证据。根因：IDE、模拟器、真机容器和审核版能力不同。
- 反例 2：AXML 指令残留。错法：微信迁移后保留 wx:if、wx:for、bindtap。对法：改为 a:if、a:for、a:key、onTap/catchTap 并真机点验。根因：支付宝模板和事件模型不是换前缀。
- 反例 3：ACSS 样式误判。错法：只看 IDE 样式正常，不查 acss、样式隔离和真机字号。对法：核页面 json、组件隔离、Android/iOS 截图。根因：容器样式隔离和端差异会放大布局问题。
- 反例 4：Component 通信照搬微信。错法：使用 triggerEvent 让父组件监听。对法：用 this.props.onChange 类回调并处理缺省。根因：支付宝组件通信机制不同。
- 反例 5：Page 参数类型错。错法：onLoad(query).id 直接数值运算或布尔判断。对法：按 string 解析、校验、兜底。根因：跳转、二维码、订阅消息参数默认字符串。
- 反例 6：setData 大对象。错法：滚动或输入时 setData 整个列表/表单。对法：只更新变化字段，分页、节流、虚拟化。根因：视图层通信成本高，低端机更明显。
- 反例 7：分包 tabBar 冲突。错法：把 tabBar 页放进分包或分包路径大小写混乱。对法：tabBar 留主包，路由表和二维码入口逐项验证。根因：包加载规则和路径规则限制。
- 反例 8：授权 scope 混乱。错法：my.getAuthCode scopes 传数组，或 auth_base/auth_user 混用。对法：scopes 用字符串，按用途拆分授权和拒绝路径。根因：scope 影响弹窗、换态、审核和用户感知。
- 反例 9：用户标识串号。错法：把 open_id 当 user_id、业务 uid 或会员 id。对法：服务端建立 user_id/open_id/会员/商家门店映射。根因：标识范围、主体和稳定性不同。
- 反例 10：接口只测 IDE。错法：my.request 在 IDE 成功就放行。对法：真机核域名白名单、证书、header、trace id 和灰度环境。根因：真机网络栈、证书和网关策略更接近线上。
- 反例 11：支付只信客户端。错法：my.tradePay 返回 9000 就改订单成功。对法：notify 验签和服务端查单裁定最终态，8000/6004 轮询。根因：客户端结果只代表调起/回流，不代表资金确认。
- 反例 12：JSAPI/H5 支付边界不清。错法：端内 WebView 内直接复用小程序支付状态。对法：区分 H5、JSAPI、小程序调起端，用短期票据和服务端订单态同步。根因：容器、登录态和回跳链路不同。
- 反例 13：订阅消息无动作。错法：页面加载即申请订阅或模板用途与业务不符。对法：用户明确动作后申请，模板、文案、落地页与业务一致。根因：2024-2026 平台审核强化授权时机和用途一致性。
- 反例 14：开放能力缺权限。错法：代码接入会员、门店、发票、生活号能力后直接提审。对法：先核开放平台能力、商家主体、类目、门店权限并准备降级。根因：开放能力受主体、类目和权限包约束。
- 反例 15：web-view 登录态断裂。错法：H5 依赖 Cookie 或任意回跳 URL。对法：白名单、短期票据、回跳验签、过期处理。根因：小程序容器与 H5 存储隔离且 URL 易被滥用。
- 反例 16：插件供应链失控。错法：只声明插件 appId，不锁版本不查权限。对法：锁版本、核权限、降级开关、真机验证。根因：插件依赖第三方发布、权限和基础库兼容。
- 反例 17：隐私审核不一致。错法：协议没写手机号、定位、剪贴板、设备信息，代码却采集。对法：采集字段、用途、弹窗、协议、埋点逐项对账。根因：2024-2026 平台隐私审核更看实际采集与声明一致。
- 反例 18：灰度无观测。错法：灰度发布后只看是否能打开首页。对法：看登录成功率、授权拒绝率、接口错误率、支付转化、web-view 错误和回滚开关。根因：灰度问题常出在账号、缓存、新旧接口和能力配置。
- 反例 19：构建产物漂移。错法：IDE 本地能跑就发布 CI/体验版产物。对法：记录构建命令、环境变量、npm/插件版本、source map、包体积和产物版本。根因：构建链、压缩、资源路径和环境注入会改变运行行为。
- 反例 20：缓存串号。错法：token/session、会员态、门店态长期存在同一 key。对法：key 带环境和用户维度，退出登录清理，换态失败不复用旧 session。根因：体验版/线上、多账号、弱网重试会放大旧缓存污染。
- 反例 21：入口参数可信。错法：分享、二维码、scheme 参数直接驱动订单、门店、活动。对法：字符串解析、白名单、签名或短码、过期兜底。根因：外部入口可被篡改、回放或污染渠道归因。
- 反例 22：原生组件层级误判。错法：弹层、fixed、scroll-view 只在 IDE 看效果。对法：低端 Android/iOS 真机录屏，核页面 json、键盘高度、表单 payload。根因：原生组件、键盘和容器层级在真机差异更明显。
- 反例 23：跨端源码假绿。错法：只看 Taro、uni-app、Mpx、Remax 源码分支正确。对法：核支付宝产物 json、axml/acss/js 和真机行为。根因：插件、条件编译和运行时映射才决定最终产物。
- 反例 24：组件库声明漏配。错法：AntD Mini import 成功就认为可用。对法：核 usingComponents/mycomponents、component2、styleIsolation 和基础库。根因：支付宝组件声明和隔离配置会影响运行时加载。
- 反例 25：WebSocket 多连接。错法：每个页面各自创建连接并重复监听。对法：全局单连接、页面订阅、幂等重连、记录回调时序。根因：前后台、弱网和页面栈会放大重复连接和消息乱序。
- 反例 26：unsupported API 被吞。错法：跨端 API proxy 捕获异常后只返回空对象。对法：保留原始错误、降级路径和兼容探测证据。根因：支付宝能力边界与其他端不同，吞错会破坏定位。
- 反例 27：前端保存 app secret。错法：为了调通开放能力，把 app secret、access_token 或手机号解密逻辑写进小程序。对法：前端只拿 authCode 或临时凭证，敏感换态和解密全部在服务端。根因：小程序包、日志和调试链路都不可信。
- 反例 28：前端裁定金额。错法：订单金额、优惠、会员价、门店价由页面参数或缓存传给支付链路。对法：前端只传商品/订单意图，金额、优惠和订单状态由服务端计算并落库。根因：入口参数和缓存可被篡改或污染。
- 反例 29：授权拒绝无路可走。错法：用户拒绝手机号、定位、相册或订阅消息后页面一直 loading。对法：拒绝态给非必要功能降级、重新授权入口和明确文案。根因：授权不是必然成功，审核也会验证拒绝路径。
- 反例 30：订阅模板和落地页不一致。错法：模板文案写优惠到账，落地页却打开空白或无对应业务。对法：模板、用户动作、发送条件、落地页、失败回调逐项对账。根因：平台审核和用户投诉都看用途一致性。
- 反例 31：API 字段直灌 UI。错法：接口把 price 从 number 改成 string 或字段缺失后页面直接崩溃。对法：请求层做 schema/兼容转换，UI 有空态、错误态和旧字段 fallback。根因：灰度、旧接口和多端版本会长期并存。
- 反例 32：WebView 回跳信任 returnUrl。错法：H5 带任意 returnUrl 或订单号回小程序就直接跳转/改态。对法：URL 白名单、短期票据、回跳验签、服务端换态。根因：端内 WebView 和 H5 参数可被构造。
- 反例 33：审核材料补不齐。错法：开发完成后才发现测试账号、商家权限、支付环境、门店数据不可复测。对法：开发前建审核路径清单，提审前用体验版和测试账号全链路录屏。根因：审核验证的是可复测业务，不是代码意图。
- 反例 34：只测 happy path 支付。错法：只测 9000 成功，取消、处理中、弱网、重复点击都没测。对法：覆盖 9000、8000、6004、取消、断网回流、查单和幂等。根因：资金链路最大风险在不确定态和重复态。

## 2024-2026 新坑速查

- 审核与隐私更严：手机号、定位、剪贴板、设备信息、相册、通讯地址、埋点都要最小化并说明用途。
- 授权时机更敏感：授权登录、用户信息、订阅消息必须绑定清晰业务动作；拒绝授权要有可用降级。
- IDE 与真机差异仍高发：网络、缓存、授权模拟、插件、web-view、支付调起以真机和体验版为准。
- 基础库兼容要下探：新 API、插件、web-view、开放能力、相册、定位、蓝牙、扫码、文件、地图可能只在高基础库可用。
- 跨端构建要看产物：Taro、uni-app、Mpx、Remax、AntD Mini 的源码配置只有结合支付宝 app/page/component json、axml/acss/js 才能作为证据。
- 商家能力受主体约束：会员、门店、券、发票、生活号、商家小程序能力受类目、资质、权限包影响。
- 支付状态更需幂等：9000、8000、6004、取消、网络错误、回流丢失都要查单、验签、幂等。
- 订阅消息审核看一致性：模板、触发动作、落地页、实际文案和业务用途要一致。
- 端内 WebView 安全抬高：URL 白名单、短期票据、回跳验签、第三方 Cookie 隔离要专项验证。
- 插件供应链风险增加：插件下架、版本变更、隐私声明、基础库兼容都会影响上线。
- 分包和包体积继续影响首屏：主包资源、字体、图片、SDK、插件膨胀要量化。
- WebSocket 更易受页面栈影响：单连接、订阅解绑、回调时序、弱网重连和后台策略都要有日志。
- unsupported API 要显式降级：API proxy 必须保留原始错误和兼容探测证据。
- 入口治理影响增长链路：分享、二维码、小程序码、scheme/URL、生活号、商家入口要防篡改、防回放、可归因、可降级。
- 本地缓存影响审核和灰度：体验版、线上版、多账号、弱网重试、退出登录都要有缓存清理和 TTL 策略。
- 灰度发布要看业务指标：错误率、支付转化、授权拒绝率、订阅到达率、web-view 错误率都要有阈值。
- 审核材料要可复测：测试账号、商家账号、支付环境、门店数据、审核路径与线上配置一致。

## 与相邻技能的边界

- 支付宝小程序/alipay-miniprogram（almp）：负责支付宝小程序原生 AXML/ACSS/SJS、Page、Component、my API、IDE 真机预览、分包、授权登录、订阅消息、开放能力、商家小程序、web-view、插件、体验版、审核与灰度排障。
- cross-platform：负责 Taro、uni-app、Mpx、Remax 等多端框架的源码组织、条件编译、构建插件和多端一致性；本技能只裁定支付宝目标和产物证据。
- 支付宝支付/alipay-pay（ali）：负责支付宝支付产品、服务端下单、notify 验签、查单、退款、资金状态、支付安全和支付产品规则；小程序只负责调起与前端证据。
- API 工程/api-engineering（api）：负责 my.request 接口契约、鉴权语义、错误码、幂等、版本兼容、灰度 API 策略。
- UI 设计实现/ui-design（u）：负责视觉层级、组件状态、空态/错误态、无障碍、设计一致性；小程序技能只判断 ACSS、原生组件与容器差异导致的落地问题。
- JavaScript/TypeScript 开发/javascript-typescript-development（jsts）：负责 JS/TS 业务逻辑、异步并发、类型、状态组织、工具函数和构建脚本；本技能只约束支付宝运行时和 my API 行为。
- Web 安全/web-security（wsec）：负责 WebView URL 安全、入口参数、票据、回跳验签、XSS、CSRF、敏感信息泄露、鉴权绕过和安全测试。
- 发布部署/release-engineering（rls）：负责体验版、审核、灰度发布、监控、告警、回滚、环境配置、构建产物和发布节奏。
- 测试验证/test-engineering（tst）：负责 compile snapshot、e2e、真机/体验版/审核回归矩阵、自动化策略、证据口径和无法验证边界。
- 代码审计/code-audit（aud）：负责修改完成前的需求对账、影响面追踪、安全质量、证据闭环和剩余风险收口。
- 数据库工程/database-engineering（db）：负责 user_id/open_id/会员映射、订单状态、幂等记录、通知日志、审计字段和数据一致性。
- 后端工程/backend-engineering（be）：负责服务端授权换态、支付下单/验签/查单、域名证书、开放能力服务端配置和日志追踪。
